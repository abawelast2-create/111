## نمط المصادقة (Authentication Pattern)

### المدير (Admin) — Session-Based

```
┌──────────┐     POST /admin/login     ┌──────────────┐
│ متصفح   │ ──────────────────────────→ │ LoginController│
│ المدير   │     username + password   │              │
└──────────┘                           └──────┬───────┘
                                              │
                                    Hash::check()
                                              │
                                    ┌─────────▼──────────┐
                                    │ Session Storage    │
                                    │ admin_id = 1       │
                                    │ admin_name = "..." │
                                    │ session_created_at │
                                    │ last_activity      │
                                    └────────────────────┘
                                              │
                                    AdminAuth Middleware
                                    يفحص session('admin_id')
                                              │
                                    كل 5 دقائق: session regenerate
                                    بعد 30 دقيقة خمول: تسجيل خروج
                                    مع remember_me: 30 يوم
```

### الموظف (Employee) — Token-Based

```
لا يوجد Session! الموظف يُعرّف عن نفسه بـ:

1. PIN (4 أرقام) → يُستخدم للدخول الأولي فقط
   PIN → findByPin() → يرجع unique_token
   
2. Token (64 حرف hex) → يُستخدم لكل العمليات
   Token في URL: /employee/attendance?token=abc123...
   Token في API POST body: { "token": "abc123..." }
   
   ⚠ Token لا يُخزن في Session أو Cookie
   ⚠ Token يظل نشطاً حتى يُعاد توليده أو يُحذف الموظف
```

## خدمات النظام (Services)

### AttendanceService

```php
// تسجيل عملية حضور/انصراف
AttendanceService::record(
    employeeId: 1,
    type: 'in',        // 'in' | 'out' | 'overtime-start' | 'overtime-end'
    lat: 24.72350,
    lon: 46.63510,
    accuracy: 15.0
): array
// Returns: ['success' => bool, 'message' => string, 'late_minutes' => int]

// الخطوات الداخلية:
// 1. التحقق من أن type صالح
// 2. فحص: هل سُجّل نفس النوع خلال 5 دقائق؟ (منع التكرار)
// 3. حساب التأخر (فقط لـ type='in'):
//    late_minutes = max(0, (now - work_start_time) / 60)
// 4. إنشاء سجل Attendance جديد
// 5. إرجاع النتيجة

// فحص النافذة الزمنية
AttendanceService::isWithinTimeWindow(
    nowTime: '08:15',
    startTime: '07:00',
    endTime: '10:00'
): bool
// يدعم النوافذ التي تمتد عبر منتصف الليل (endTime < startTime)
```

### GeofenceService

```php
// حساب المسافة بين نقطتين (صيغة Haversine)
GeofenceService::calculateDistance(
    lat1: 24.72350, lon1: 46.63510,  // موقع الموظف
    lat2: 24.72341, lon2: 46.63524   // موقع الفرع
): float
// Returns: المسافة بالمتر (مثل 18.5)

// Haversine Formula:
// a = sin²(Δlat/2) + cos(lat1) × cos(lat2) × sin²(Δlon/2)
// c = 2 × asin(√a)
// d = R × c  (R = 6,371,000 meters)

// فحص: هل الموظف داخل الجيوفينس؟
GeofenceService::isWithinGeofence(
    empLat: 24.72350,
    empLon: 46.63510,
    branchId: 1
): array
// Returns:
// ['allowed' => true, 'distance' => 18, 'radius' => 500, 'message' => '...']

// أولوية الفحص:
// 1. إذا branchId موجود → يستخدم إحداثيات الفرع
// 2. إذا لا → يستخدم الإعدادات العامة (جدول settings)
// 3. إذا لم يُحدد موقع عمل → allowed = true (مسموح دائماً)
```

## Middleware (الطبقات الوسيطة)

### AdminAuth — حماية لوحة التحكم

```php
// السلوك:
handle(Request $request, Closure $next) {
    // 1. هل admin_id موجود في Session؟
    if (!session('admin_id')) → redirect('/admin/login') أو 401 JSON

    // 2. تجديد الجلسة كل 5 دقائق (حماية ضد session hijacking):
    if (time() - session('session_created_at') > 300) {
        session()->regenerate();
        session(['session_created_at' => time()]);
    }

    // 3. فحص الخمول:
    // - بدون remember: 30 دقيقة
    // - مع remember: 30 يوم
    if (timeout expired) → flush session → redirect login

    // 4. تحديث last_activity:
    session(['last_activity' => time()]);
}
```

### RateLimiter — حد الطلبات

```php
// المعاملات:
$maxRequests = 60;  // عدد الطلبات المسموح
$prefix = 'api';    // مفتاح التخزين

// السلوك:
// - يستخدم Cache لتخزين العدّاد لكل IP
// - key = "rate_limit:{prefix}:{ip}"
// - TTL = 60 ثانية
// - عند تجاوز الحد → 429 Too Many Requests + Retry-After header
```

### SecurityHeaders — رؤوس الأمان

```php
// تُضاف لكل استجابة:
'X-Content-Type-Options' => 'nosniff'           // منع تخمين نوع المحتوى
'X-Frame-Options' => 'SAMEORIGIN'               // منع Clickjacking
'X-XSS-Protection' => '1; mode=block'           // حماية XSS
'Referrer-Policy' => 'strict-origin-when-cross-origin'
'Permissions-Policy' => 'geolocation=(self)...'  // تقييد APIs
'Alt-Svc' => 'clear'                            // منع protocol coercion
```

## التخزين المؤقت (Caching)

### Setting Cache

```php
// Setting::loadAll() يستخدم static cache:
protected static ?array $cache = null;

// عند أول استدعاء في الطلب:
$cache = Setting::all()->pluck('setting_value', 'setting_key')->toArray();

// باقي الاستدعاءات في نفس الطلب:
return $cache; // لا يذهب لقاعدة البيانات

// يُنعش عند:
Setting::loadAll(refresh: true);
```
