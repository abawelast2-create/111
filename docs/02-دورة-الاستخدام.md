## المرحلة 1: تسجيل الحضور (Check-In)

### 1.1 — الموظف يفتح الرابط

**الموظف** يفتح `mycorner.site` → النظام يعيد التوجيه إلى `/employee`

**ما يحدث تقنياً:**
```
GET mycorner.site → 302 Redirect → /employee
GET /employee → يُعرض نموذج إدخال PIN
```

### 1.2 — إدخال رمز PIN

الموظف يدخل `3847` ويضغط "دخول"

**ما يحدث في الكود:**
```
POST /employee → EmployeeController@authByPin
  → validates: pin = "3847" (string, size:4)
  → Employee::findByPin("3847")
    → SELECT * FROM employees WHERE pin = '3847' AND is_active = 1 AND deleted_at IS NULL
  → Redirect → /employee/attendance?token=a7b3c9d1...
```

**لا تتغير أي بيانات في قاعدة البيانات في هذه الخطوة.**

### 1.3 — صفحة الحضور تُحمّل

```
GET /employee/attendance?token=a7b3c9d1...
  → Employee::findByToken("a7b3c9d1...")
  → يجلب سجلات حضور اليوم
  → يجلب جدول مواعيد الفرع
  → يعرض الخريطة + الرادار + الأزرار
```

**قاعدة البيانات: استعلامات قراءة فقط:**
```sql
SELECT * FROM employees WHERE unique_token = 'a7b3c9d1...' AND is_active = 1;
SELECT * FROM branches WHERE id = 1 AND is_active = 1;
SELECT * FROM attendances WHERE employee_id = 1 AND attendance_date = '2026-03-15' ORDER BY timestamp;
```

### 1.4 — المتصفح يحدد الموقع الجغرافي

JavaScript يحصل على GPS:
```javascript
navigator.geolocation.getCurrentPosition(...)
// النتيجة: latitude = 24.72350, longitude = 46.63510, accuracy = 15
```

**ما يحدث في GeofenceService:**
```
المسافة = Haversine(24.72350, 46.63510, 24.72341, 46.63524)
       = ~18 متر  ← أقل من 500 متر
       = ✅ داخل النطاق
```

### 1.5 — الموظف يضغط "تسجيل الحضور"

**طلب API:**
```
POST /api/check-in
Body: {
    token: "a7b3c9d1...",
    latitude: 24.72350000,
    longitude: 46.63510000,
    accuracy: 15.00
}
```

**ما يحدث في الكود خطوة بخطوة:**

```
1. CheckInController يستقبل الطلب
2. يتحقق من Token → يجد الموظف "أحمد محمد"
3. يجلب جدول الفرع → check_in_start: 07:00, check_in_end: 10:00
4. الوقت الحالي 08:15 → ✅ ضمن النافذة
5. يتحقق من الموقع → 18 متر ≤ 500 متر → ✅ داخل النطاق
6. يستدعي AttendanceService::record(...)
   6a. يتحقق: هل سجّل خلال آخر 5 دقائق؟ → لا
   6b. يحسب التأخر: work_start = 08:00, الآن = 08:15 → 15 دقيقة تأخر
   6c. يُنشئ السجل
```

**⭐ ما يتغير في قاعدة البيانات:**

```
┌ attendances (سجل جديد) ──────────────────┐
│ id: 1                                     │
│ employee_id: 1          ← رقم الموظف      │
│ type: "in"              ← حضور            │
│ timestamp: 2026-03-15 08:15:30            │
│ attendance_date: 2026-03-15               │
│ late_minutes: 15        ← 15 دقيقة تأخر   │
│ latitude: 24.72350000   ← موقع الموظف     │
│ longitude: 46.63510000                    │
│ location_accuracy: 15.00 ← دقة GPS بالمتر │
│ ip_address: "188.XX.XX.XX"               │
│ user_agent: "Mozilla/5.0..."             │
│ notes: NULL                              │
│ created_at: 2026-03-15 08:15:30          │
│ updated_at: 2026-03-15 08:15:30          │
└───────────────────────────────────────────┘
```

**الاستجابة:**
```json
{
    "success": true,
    "message": "تم تسجيل الدخول بنجاح",
    "late_minutes": 15,
    "employee_name": "أحمد محمد",
    "timestamp": "2026-03-15 08:15:30",
    "distance": 18
}
```

### 1.6 — لو الموظف خارج النطاق

```
المسافة = 1200 متر > 500 متر → ❌ مرفوض
```

**الاستجابة:**
```json
{
    "success": false,
    "message": "أنت خارج نطاق العمل! المسافة: 1200 متر (الحد المسموح: 500 متر)",
    "distance": 1200
}
```
**لا تتغير قاعدة البيانات.**

### 1.7 — لو الموظف خارج النافذة الزمنية

الوقت 11:30 والنافذة تنتهي 10:00 → ❌ مرفوض

```json
{
    "success": false,
    "message": "وقت تسجيل الدخول المسموح به: 07:00 - 10:00. الوقت الحالي: 11:30"
}
```
**لا تتغير قاعدة البيانات.**

## المرحلة 7: دخول المدير وعرض التقارير

### 7.1 — تسجيل دخول المدير

```
POST /admin/login
Body: { username: "admin", password: "..." }
```

**ما يحدث:**
1. التحقق من قفل IP (brute force):
```sql
SELECT COUNT(*) FROM login_attempts
WHERE ip_address = '188.XX.XX.XX'
AND attempted_at > NOW() - INTERVAL 10 MINUTE;
-- إذا كانت النتيجة ≥ 5 → مقفل
```

2. التحقق من كلمة المرور:
```php
Hash::check($request->password, $admin->password_hash)
```

**⭐ ما يتغير في قاعدة البيانات:**

### 7.2 — لوحة القيادة (Dashboard)

```
GET /admin/dashboard
```

**الاستعلامات التي تُنفّذ:**
```sql
-- 1. إحصائيات اليوم
SELECT
  COUNT(DISTINCT CASE WHEN type='in' THEN employee_id END) as checked_in,
  COUNT(DISTINCT CASE WHEN type='out' THEN employee_id END) as checked_out,
  (SELECT COUNT(*) FROM employees WHERE is_active=1 AND deleted_at IS NULL) as total
FROM attendances WHERE attendance_date = '2026-03-15';

-- 2. عدد الفروع النشطة
SELECT COUNT(*) FROM branches WHERE is_active = 1;

-- 3. آخر 15 تسجيلاً
SELECT a.*, e.name, e.job_title, b.name as branch_name
FROM attendances a
JOIN employees e ON a.employee_id = e.id
LEFT JOIN branches b ON e.branch_id = b.id
ORDER BY a.timestamp DESC LIMIT 15;

-- 4. الغائبون
SELECT e.name, e.job_title, b.name as branch_name
FROM employees e
LEFT JOIN branches b ON e.branch_id = b.id
WHERE e.is_active = 1 AND e.deleted_at IS NULL
AND e.id NOT IN (
    SELECT DISTINCT employee_id FROM attendances
    WHERE attendance_date = '2026-03-15' AND type = 'in'
);
```

**لا تتغير قاعدة البيانات — استعلامات قراءة فقط.**

### 7.3 — تقرير التأخر

```
GET /admin/late-report?from=2026-03-01&to=2026-03-15
```

**الاستعلامات:**
```sql
-- سجلات التأخر
SELECT a.*, e.name, b.name as branch_name
FROM attendances a
JOIN employees e ON a.employee_id = e.id
LEFT JOIN branches b ON e.branch_id = b.id
WHERE a.type = 'in'
AND a.late_minutes > 0
AND a.attendance_date BETWEEN '2026-03-01' AND '2026-03-15'
ORDER BY a.timestamp DESC;
```

**المعالجة في الكود:**
```
لكل موظف:
  - عدد مرات التأخر = count(سجلات التأخر)
  - إجمالي الدقائق = sum(late_minutes)
  - المعدل = إجمالي / عدد المرات
  - التقييم:
    • ≤ 5 دقائق متوسط ← "ممتاز" 🟢
    • ≤ 15 دقيقة متوسط ← "مقبول" 🟡
    • > 15 دقيقة متوسط ← "ضعيف" 🔴
```

### 7.4 — تقرير الرسوم البيانية

```
GET /admin/report-charts?from=2026-02-15&to=2026-03-15
```

**يجلب 4 مجموعات بيانات:**

```
1. الحضور اليومي:
   تاريخ → عدد تسجيلات الدخول
   {2026-03-01: 45, 2026-03-02: 42, ...}

2. توزيع الأنواع:
   {in: 300, out: 295, overtime-start: 15, overtime-end: 14}

3. مقارنة الفروع:
   {الفرع الرئيسي: 200, فرع الشمال: 100}

4. مؤشر التأخر:
   تاريخ → عدد المتأخرين
   {2026-03-01: 5, 2026-03-02: 3, ...}
```

```

```
┌ employees (تحديث) ──────────────────────┐
