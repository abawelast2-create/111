# 🚀 دليل التثبيت والنشر

> هذا الدليل يشرح كيفية إعداد النظام محلياً وعلى خادم الإنتاج.

---

## المتطلبات

| المكوّن | الإصدار الأدنى | ملاحظات |
|---------|---------------|---------|
| PHP | 8.1+ | مع الإضافات: PDO, PDO_MySQL, mbstring, json, openssl, tokenizer, ctype, fileinfo |
| Composer | 2.x | لإدارة اعتمادات PHP |
| MySQL / MariaDB | 8.0+ / 10.5+ | ترميز utf8mb4_unicode_ci |
| Node.js | 18+ | لبناء الأصول (Vite) — للتطوير فقط |
| خادم الويب | Nginx أو Apache | مع دعم URL Rewriting |

## التثبيت المحلي (التطوير)

### 1. استنساخ المشروع

```bash
git clone <repository-url> attendance-laravel
cd attendance-laravel
```

### 2. تثبيت الاعتمادات

```bash
composer install
npm install
```

### 3. إعداد البيئة

```bash
cp .env.example .env
php artisan key:generate
```

عدّل ملف `.env`:

```env
APP_NAME="نظام الحضور والانصراف"
APP_ENV=local
APP_DEBUG=true
APP_URL=http://localhost:8000

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=attendance_db
DB_USERNAME=root
DB_PASSWORD=

SESSION_DRIVER=file
CACHE_DRIVER=file
```

### 4. إنشاء قاعدة البيانات

```bash
mysql -u root -e "CREATE DATABASE attendance_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
```

### 5. تشغيل الهجرات والبذور

```bash
php artisan migrate
php artisan db:seed
```

> **البيانات الأولية:** يُنشئ 5 فروع + 30+ موظف + حساب مدير افتراضي + 16 إعداد نظام.

### 6. تشغيل خادم التطوير

```bash
php artisan serve
# يعمل على http://localhost:8000
```

## متغيرات البيئة (`.env`)

### إعدادات التطبيق

| المتغير | الوصف | القيمة الافتراضية |
|---------|-------|-------------------|
| `APP_NAME` | اسم النظام | `Laravel` |
| `APP_ENV` | البيئة: `local` / `production` | `local` |
| `APP_DEBUG` | وضع التصحيح (false في الإنتاج) | `true` |
| `APP_URL` | رابط الموقع الكامل | `http://localhost` |
| `APP_KEY` | مفتاح التشفير (يُولّد تلقائياً) | — |

### إعدادات قاعدة البيانات

| المتغير | الوصف | القيمة الافتراضية |
|---------|-------|-------------------|
| `DB_CONNECTION` | نوع قاعدة البيانات | `mysql` |
| `DB_HOST` | عنوان الخادم | `127.0.0.1` |
| `DB_PORT` | المنفذ | `3306` |
| `DB_DATABASE` | اسم قاعدة البيانات | `laravel` |
| `DB_USERNAME` | اسم المستخدم | `root` |
| `DB_PASSWORD` | كلمة المرور | — |

### إعدادات الجلسة والتخزين المؤقت

| المتغير | الوصف | القيمة الافتراضية |
|---------|-------|-------------------|
| `SESSION_DRIVER` | مخزن الجلسات | `file` |
| `SESSION_LIFETIME` | مدة الجلسة بالدقائق | `120` |
| `CACHE_DRIVER` | مخزن التخزين المؤقت | `file` |

## النشر على خادم الإنتاج (VPS)

### 1. متطلبات الخادم

```
نظام التشغيل: Ubuntu 22.04+
خادم الويب:   Nginx + PHP 8.3-FPM
قاعدة البيانات: MariaDB 10.5+
SSL:           Let's Encrypt (certbot)
```

### 2. رفع الملفات

```bash
# عبر Git
cd /var/www
git clone <repository-url> attendance
cd attendance

# أو عبر SFTP/SCP
scp -r ./attendance-laravel root@your-server:/var/www/attendance
```

### 3. تثبيت الاعتمادات (إنتاج)

```bash
cd /var/www/attendance
composer install --optimize-autoloader --no-dev
```

### 4. إعداد البيئة

```bash
cp .env.example .env
php artisan key:generate
```

عدّل `.env` للإنتاج:

```env
APP_ENV=production
APP_DEBUG=false
APP_URL=https://yourdomain.com

DB_DATABASE=attendance_prod
DB_USERNAME=db_user
DB_PASSWORD=strong_password_here

SESSION_DRIVER=file
SESSION_SECURE_COOKIE=true
```

### 5. الصلاحيات

```bash
chown -R www-data:www-data /var/www/attendance
chmod -R 755 /var/www/attendance
chmod -R 775 storage bootstrap/cache
```

### 6. تشغيل الهجرات

```bash
php artisan migrate --force
php artisan db:seed  # فقط في أول تثبيت
```

### 7. تحسين الأداء

```bash
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

### 8. إعداد Nginx

```nginx
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl;
    server_name yourdomain.com www.yourdomain.com;
    root /var/www/attendance/public;
    index index.php;

    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/run/php/php8.3-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }
}
```

### 9. شهادة SSL

```bash
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
```

## النشر على استضافة مشتركة (Shared Hosting)

### الإعداد

1. ارفع جميع الملفات إلى جذر الاستضافة
2. استخدم `public/index.hosting.php` كنقطة دخول بديلة:
   - انسخ محتواه إلى `public_html/index.php`
   - أو اضبط Document Root على مجلد `public/`
3. أنشئ قاعدة البيانات من cPanel
4. عدّل `.env` بمعلومات الاتصال

## بعد التثبيت — خطوات إلزامية

1. **غيّر كلمة المرور** من الإعدادات ← تبويب كلمة المرور
2. **فعّل المصادقة الثنائية (2FA)** من الإعدادات ← المصادقة الثنائية
3. **حدّد موقع العمل** من الإعدادات ← تبويب الموقع الجغرافي
4. **اضبط أوقات الدوام** من الإعدادات ← تبويب المواعيد
5. **أضف الفروع** من صفحة إدارة الفروع
6. **أضف الموظفين** من صفحة إدارة الموظفين
7. **أرسل الروابط** لكل موظف عبر واتساب

## النسخ الاحتياطي

### النسخ الاحتياطي من لوحة التحكم

1. اذهب إلى **"النسخ الاحتياطي"** في لوحة المدير
2. اضغط **"إنشاء نسخة احتياطية"**
3. يُحفظ الملف في `storage/app/backups/`
4. يمكنك تحميل أو حذف النسخ القديمة
5. النظام يحتفظ بآخر 30 نسخة تلقائياً

### النسخ الاحتياطي اليدوي

```bash
# نسخ قاعدة البيانات
mysqldump -u root -p attendance_db > backup_$(date +%Y%m%d).sql

# نسخ كل المشروع
tar -czf attendance_backup_$(date +%Y%m%d).tar.gz /var/www/attendance
```

### الاستعادة

```bash
# استعادة من النسخة
mysql -u root -p attendance_db < backup_20260315.sql
```

أو من لوحة التحكم: اذهب إلى النسخ الاحتياطي → اختر نسخة → استعادة.

## استكشاف الأخطاء

| المشكلة | الحل |
|---------|------|
| صفحة بيضاء أو خطأ 500 | تحقق من `storage/logs/laravel.log` وصلاحيات `storage/` |
| خطأ اتصال قاعدة البيانات | تحقق من بيانات `.env` وأن خدمة MySQL تعمل |
| 419 Page Expired | مسح كاش الكوكيز أو تشغيل `php artisan config:cache` |
| خطوط عربية غير ظاهرة | تحقق من وجود ملفات `public/assets/fonts/tajawal/` |
| CSRF token mismatch | تأكد من وجود `@csrf` في النماذج و`EncryptCookies` في middleware |
