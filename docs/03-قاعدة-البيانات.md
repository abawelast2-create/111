# 🗄️ توثيق قاعدة البيانات — شرح كل جدول وكل خلية

> يشرح هذا المستند كل جدول في قاعدة البيانات، كل عمود فيه، ومتى وكيف يتغير.

---

## نظرة عامة

```
قاعدة البيانات: MariaDB (MySQL)
ترميز: utf8mb4_unicode_ci
عدد الجداول: 22+ (بما فيها الجداول المساعدة)
```

### خريطة العلاقات (الجداول الأساسية)

```
branches ◄───────────────── employees
                               │
                    ┌──────────┼──────────────┬────────────────┐
                    │          │              │                │
              attendances   leaves     known_devices    tampering_cases
                                                              │
admins ◄─────── leaves.approved_by                     secret_reports
   │
   └── audit_log.admin_id

settings (مستقل)
login_attempts (مستقل)
personal_access_tokens (Laravel Sanctum - مستقل)
```

> يخزن بيانات فروع العمل (المواقع الجغرافية ومواعيد الدوام)

|--------|-------|-------------------|-------|------|-------|

| `geofence_radius` | int | 500 | ✅ | ❌ | نصف قطر السياج الجغرافي بالمتر |
| `work_start_time` | time | 08:00:00 | ✅ | ❌ | بداية الدوام الرسمي |
| `work_end_time` | time | 16:00:00 | ✅ | ❌ | نهاية الدوام الرسمي |
| `check_in_start_time` | time | 07:00:00 | ✅ | ❌ | أول وقت مسموح لتسجيل الحضور |
| `check_in_end_time` | time | 10:00:00 | ✅ | ❌ | آخر وقت مسموح لتسجيل الحضور |
| `check_out_start_time` | time | 15:00:00 | ✅ | ❌ | أول وقت مسموح لتسجيل الانصراف |
| `check_out_end_time` | time | 20:00:00 | ✅ | ❌ | آخر وقت مسموح لتسجيل الانصراف |
| `checkout_show_before` | int | 30 | ✅ | ❌ | يُظهر زر الانصراف قبل X دقيقة |
| `allow_overtime` | tinyint(1) | 1 | ✅ | ❌ | هل العمل الإضافي مسموح (0/1) |
| `overtime_start_after` | int | 60 | ✅ | ❌ | يبدأ الإضافي بعد X دقيقة من الانصراف |
| `overtime_min_duration` | int | 30 | ✅ | ❌ | الحد الأدنى لمدة العمل الإضافي بالدقائق |
| `is_active` | tinyint(1) | 1 | ✅ | ❌ | هل الفرع نشط (0/1) |
| `created_at` | timestamp | NULL | ❌ | ❌ | تاريخ الإنشاء |
| `updated_at` | timestamp | NULL | ❌ | ❌ | تاريخ آخر تحديث |

### متى تتغير الخلايا؟

| الحدث | الأعمدة المتأثرة |
|-------|-----------------|
| المدير يُنشئ فرعاً | كل الأعمدة تُملأ + `created_at`, `updated_at` |
| المدير يُعدّل إعدادات الفرع | الأعمدة المُعدّلة + `updated_at` |
| المدير يُعطّل فرعاً | `is_active` → 0 + `updated_at` |
| المدير يُعيد تفعيل فرع | `is_active` → 1 + `updated_at` |

### مثال عملي

```
الحالة الأولى: إنشاء الفرع
  name: "الفرع الرئيسي"
  latitude: 24.72341500
  longitude: 46.63524100
  geofence_radius: 500
  is_active: 1
  created_at: 2026-03-15 10:00:00

الحالة الثانية: المدير يُقلّص نطاق GPS من 500 إلى 300 متر
  geofence_radius: 500 → 300  ✏️
  updated_at: 2026-03-15 10:00:00 → 2026-03-16 14:30:00  ✏️
  (باقي الأعمدة: لا تتغير)

الحالة الثالثة: المدير يُعطّل الفرع
  is_active: 1 → 0  ✏️
  updated_at: → 2026-04-01 09:00:00  ✏️
```

## 2. جدول `admins` — المديرون

> يخزن بيانات مديري النظام (لوحة التحكم)

| العمود | النوع | القيمة الافتراضية | مطلوب | فريد | الوصف |
|--------|-------|-------------------|-------|------|-------|
| `id` | bigint unsigned | auto_increment | ✅ | ✅ | المعرّف الفريد |
| `username` | varchar(50) | — | ✅ | ✅ | اسم المستخدم للدخول |
| `password_hash` | varchar(255) | — | ✅ | ❌ | كلمة المرور مُشفّرة بـ bcrypt |
| `full_name` | varchar(255) | — | ✅ | ❌ | الاسم الكامل |
| `last_login` | datetime | NULL | ❌ | ❌ | تاريخ آخر تسجيل دخول |
| `created_at` | timestamp | NULL | ❌ | ❌ | تاريخ الإنشاء |
| `updated_at` | timestamp | NULL | ❌ | ❌ | تاريخ آخر تحديث |

### متى تتغير الخلايا؟

| الحدث | الأعمدة المتأثرة |
|-------|-----------------|
| إنشاء حساب المدير | كل الأعمدة + `created_at`, `updated_at` |
| تسجيل دخول ناجح | `last_login` → الوقت الحالي + `updated_at` |
| تغيير كلمة المرور | `password_hash` → bcrypt جديد + `updated_at` |

### ملاحظة أمنية
- `password_hash` يُخزَّن بتشفير **bcrypt** (أحادي الاتجاه — لا يمكن فك تشفيره)
- لا تُخزَّن كلمة المرور الأصلية أبداً
- كل مرة تتغير كلمة المرور → hash جديد مختلف تماماً حتى لو كانت نفس الكلمة

## 3. جدول `employees` — الموظفون

> الجدول الأساسي — يخزِن بيانات كل موظف

| العمود | النوع | القيمة الافتراضية | مطلوب | فريد | الوصف |
|--------|-------|-------------------|-------|------|-------|
| `id` | bigint unsigned | auto_increment | ✅ | ✅ | المعرّف الفريد |
| `name` | varchar(255) | — | ✅ | ❌ | اسم الموظف الكامل |
| `job_title` | varchar(255) | — | ✅ | ❌ | المسمى الوظيفي |
| `pin` | varchar(10) | — | ✅ | ✅ | رمز PIN (4 أرقام) للدخول |
| `pin_changed_at` | timestamp | NULL | ❌ | ❌ | تاريخ آخر تغيير للـ PIN |
| `phone` | varchar(20) | NULL | ❌ | ❌ | رقم الهاتف |
| `unique_token` | varchar(64) | — | ✅ | ✅ | رمز سري للروابط (64 حرف hex) |
| `branch_id` | bigint unsigned | NULL | ❌ | ❌ | FK → branches.id |
| `device_fingerprint` | varchar(64) | NULL | ❌ | ❌ | بصمة الجهاز المُسجّل |
| `device_registered_at` | timestamp | NULL | ❌ | ❌ | تاريخ ربط الجهاز |
| `device_bind_mode` | tinyint | 0 | ✅ | ❌ | وضع ربط الجهاز (0/1/2) |
| `security_level` | int | 2 | ✅ | ❌ | مستوى الأمان |
| `is_active` | tinyint(1) | 1 | ✅ | ❌ | هل الحساب نشط |
| `deleted_at` | timestamp | NULL | ❌ | ❌ | تاريخ الحذف الناعم |
| `created_at` | timestamp | NULL | ❌ | ❌ | تاريخ الإنشاء |
| `updated_at` | timestamp | NULL | ❌ | ❌ | تاريخ آخر تحديث |

### أوضاع ربط الجهاز (`device_bind_mode`)

| القيمة | الوضع | السلوك |
|--------|-------|--------|
| **0** | حر (Free) | لا يوجد ربط — الموظف يدخل من أي جهاز |
| **1** | صارم (Strict) | فقط الجهاز المسجّل — أي جهاز آخر يُرفض ويُسجَّل كتلاعب |
| **2** | مراقبة (Monitor) | يسمح بأي جهاز لكن يُسجّل تنبيه عند استخدام جهاز مختلف |

### متى تتغير الخلايا؟

| الحدث | الأعمدة المتأثرة |
|-------|-----------------|
| إنشاء موظف | كل الأعمدة الأساسية + PIN وToken مُولّدان تلقائياً |
| أول دخول من جهاز (وضع ≠ 0) | `device_fingerprint` + `device_registered_at` + `updated_at` |
| المدير يُعيد توليد PIN | `pin` → رقم جديد + `pin_changed_at` + `updated_at` |
| المدير يُعيد ضبط الجهاز | `device_fingerprint` → NULL + `device_registered_at` → NULL + `updated_at` |
| المدير يُعدّل بيانات الموظف | الأعمدة المُعدّلة + `updated_at` |
| المدير يحذف الموظف | `deleted_at` → الوقت الحالي (الموظف يبقى في DB) |
| المدير يسترجع الموظف | `deleted_at` → NULL |
| المدير يُعطّل الموظف | `is_active` → 0 + `updated_at` |

### كيف يُولَّد PIN و Token؟

```
PIN:
  - random_int(0, 9999) → مثل 847
  - str_pad → "0847" (دائماً 4 أرقام)
  - يتحقق أنه فريد (لا يتكرر مع أي موظف حتى المحذوفين)

Token:
  - bin2hex(random_bytes(32)) → 64 حرف hex
  - مثل: "a7b3c9d1e5f2..."
  - يتحقق أنه فريد
```

### ماذا يحدث عند حذف الموظف (Soft Delete)؟

```
قبل الحذف:
  deleted_at: NULL    → الموظف يظهر في النظام ويمكنه الحضور

بعد الحذف:
  deleted_at: 2026-03-15 15:00:00  → الموظف مخفي
  - لا يظهر في قوائم الموظفين
  - لا يمكنه الدخول بـ PIN
  - لا يمكنه الحضور بـ Token
  - سجلاته السابقة (attendances) تبقى موجودة

بعد الاسترجاع:
  deleted_at: NULL    → يعود كما كان
```

## 4. جدول `attendances` — سجلات الحضور

> **الجدول الأهم** — يخزِن كل عملية حضور/انصراف/إضافي

| العمود | النوع | القيمة الافتراضية | مطلوب | فريد | الوصف |
|--------|-------|-------------------|-------|------|-------|
| `id` | bigint unsigned | auto_increment | ✅ | ✅ | المعرّف الفريد |
| `employee_id` | bigint unsigned | — | ✅ | ❌ | FK → employees.id (إذا حُذف الموظف → تحذف سجلاته) |
| `type` | enum | — | ✅ | ❌ | نوع التسجيل: 'in' / 'out' / 'overtime-start' / 'overtime-end' |
| `timestamp` | datetime | — | ✅ | ❌ | الوقت الدقيق للتسجيل |
| `attendance_date` | date | — | ✅ | ❌ | تاريخ اليوم (للفلترة والتقارير) |
| `late_minutes` | int | 0 | ✅ | ❌ | دقائق التأخر (فقط لـ type=in) |
| `latitude` | decimal(10,8) | — | ✅ | ❌ | خط عرض موقع الموظف وقت التسجيل |
| `longitude` | decimal(11,8) | — | ✅ | ❌ | خط طول موقع الموظف وقت التسجيل |
| `location_accuracy` | decimal(5,2) | NULL | ❌ | ❌ | دقة GPS بالمتر (15.00 = ممتاز) |
| `ip_address` | varchar(45) | NULL | ❌ | ❌ | عنوان IP للموظف |
| `user_agent` | text | NULL | ❌ | ❌ | معلومات المتصفح/الجهاز |
| `notes` | text | NULL | ❌ | ❌ | ملاحظات إضافية |
| `created_at` | timestamp | NULL | ❌ | ❌ | تاريخ إنشاء السجل |
| `updated_at` | timestamp | NULL | ❌ | ❌ | تاريخ آخر تحديث |

### الفهارس (Indexes)
```sql
INDEX(employee_id, timestamp)  -- للبحث السريع عن سجلات موظف معين
INDEX(type, timestamp)         -- للبحث حسب النوع
INDEX(attendance_date)         -- للبحث حسب التاريخ
```

### أنواع التسجيل (`type`)

| القيمة | المعنى | متى يُنشأ | ماذا يحدث لـ `late_minutes` |
|--------|--------|-----------|---------------------------|
| `in` | تسجيل حضور | عند الضغط على "تسجيل الحضور" | **يُحسب:** (الوقت الحالي - work_start_time) بالدقائق. 0 إذا لم يتأخر |
| `out` | تسجيل انصراف | عند الضغط على "تسجيل الانصراف" | دائماً **0** |
| `overtime-start` | بداية عمل إضافي | عند الضغط على "بدء العمل الإضافي" | دائماً **0** |
| `overtime-end` | نهاية عمل إضافي | عند الضغط على "إنهاء العمل الإضافي" | دائماً **0** |

### كيف يُحسب التأخر (`late_minutes`)؟

```
work_start_time = 08:00 (من إعدادات الفرع)
timestamp = 08:15:30 (وقت التسجيل الفعلي)

الفرق = 08:15 - 08:00 = 15 دقيقة
late_minutes = 15

لو سجّل 07:55:
  الفرق = 07:55 - 08:00 = -5 (سالب = مبكر)
  late_minutes = max(0, -5) = 0 ← لا يوجد تأخر
```

### متى يُنشأ سجل جديد؟

| الحدث | الشروط التي تُفحص | النتيجة |
|-------|-------------------|---------|
| الموظف يضغط "حضور" | ✅ Token صالح + ✅ ضمن النافذة الزمنية + ✅ داخل الجيوفينس + ✅ لم يُسجّل خلال 5 دقائق | سجل جديد type=in |
| الموظف يضغط "انصراف" | ✅ Token + ✅ نافذة انصراف + ✅ جيوفينس + ✅ لم يُسجّل خلال 5 دقائق | سجل جديد type=out |
| الموظف يضغط "بدء إضافي" | ✅ Token + ✅ يوجد checkout اليوم + ✅ الإضافي مسموح + ✅ جيوفينس | سجل جديد type=overtime-start |
| الموظف يضغط "إنهاء إضافي" | ✅ Token + ✅ يوجد overtime-start اليوم + ✅ المدة ≥ الحد الأدنى | سجل جديد type=overtime-end |

### متى يُمكن حذف سجل؟

- فقط **المدير** يمكنه حذف سجلات الحضور
- يُسجَّل الحذف في `audit_log`
- الحذف **نهائي** (ليس soft delete)

## 6. جدول `login_attempts` — محاولات الدخول الفاشلة

> يخزن محاولات الدخول الفاشلة لحماية النظام من هجمات تخمين كلمة المرور

| العمود | النوع | القيمة الافتراضية | مطلوب | فريد | الوصف |
|--------|-------|-------------------|-------|------|-------|
| `id` | bigint unsigned | auto_increment | ✅ | ✅ | المعرّف |
| `ip_address` | varchar(45) | — | ✅ | ❌ | عنوان IP للمحاوِل |
| `username` | varchar(50) | NULL | ❌ | ❌ | اسم المستخدم المُستخدَم |
| `attempted_at` | datetime | — | ✅ | ❌ | وقت المحاولة |

### الفهارس
```sql
INDEX(ip_address, attempted_at) -- للبحث السريع عن محاولات IP معين
```

### آلية الحماية (Brute Force Protection)

```
1. عند كل محاولة دخول فاشلة → يُسجّل سطر جديد
2. عند كل محاولة دخول → يُفحص:
   SELECT COUNT(*) FROM login_attempts
   WHERE ip_address = 'xxx' AND attempted_at > NOW() - 10 minutes
3. إذا العدد ≥ 5 → IP مقفول لمدة 10 دقائق
4. تُحذف السجلات القديمة (> 60 دقيقة) دورياً
```

## 7. جدول `audit_log` — سجل المراجعة

> يُسجّل كل إجراء يقوم به المدير (للمراجعة والمساءلة)

| العمود | النوع | القيمة الافتراضية | مطلوب | فريد | الوصف |
|--------|-------|-------------------|-------|------|-------|
| `id` | bigint unsigned | auto_increment | ✅ | ✅ | المعرّف |
| `admin_id` | bigint unsigned | NULL | ❌ | ❌ | رقم المدير (أو NULL للنظام) |
| `action` | varchar(50) | — | ✅ | ❌ | نوع الإجراء |
| `details` | text | NULL | ❌ | ❌ | تفاصيل الإجراء |
| `target_id` | bigint unsigned | NULL | ❌ | ❌ | رقم العنصر المتأثر |
| `ip_address` | varchar(45) | NULL | ❌ | ❌ | عنوان IP |
| `created_at` | datetime | — | ✅ | ❌ | وقت الإجراء |

### أنواع الإجراءات المسجّلة (`action`)

| القيمة | متى يُسجّل |
|--------|-----------|
| `login` | تسجيل دخول ناجح للمدير |
| `logout` | تسجيل خروج المدير |
| `create_employee` | إنشاء موظف جديد |
| `update_employee` | تعديل بيانات موظف |
| `delete_employee` | حذف موظف |
| `restore_employee` | استرجاع موظف محذوف |

## استعلامات SQL شائعة

### إحصائيات اليوم

```sql
-- عدد الحاضرين اليوم
SELECT COUNT(DISTINCT employee_id)
FROM attendances
WHERE attendance_date = CURDATE()
AND type = 'in';

-- عدد المنصرفين اليوم
SELECT COUNT(DISTINCT employee_id)
FROM attendances
WHERE attendance_date = CURDATE()
AND type = 'out';

-- الغائبون اليوم
SELECT e.name, e.job_title, b.name AS branch
FROM employees e
LEFT JOIN branches b ON e.branch_id = b.id
WHERE e.is_active = 1
AND e.deleted_at IS NULL
AND e.id NOT IN (
    SELECT DISTINCT employee_id
    FROM attendances
    WHERE attendance_date = CURDATE()
    AND type = 'in'
);
```

### تقارير الموظفين

```sql
-- تقرير حضور موظف لفترة زمنية
SELECT attendance_date, type, timestamp, late_minutes, latitude, longitude
FROM attendances
WHERE employee_id = ?
AND attendance_date BETWEEN ? AND ?
ORDER BY timestamp;

-- إجمالي التأخر لكل موظف هذا الشهر
SELECT e.name,
       COUNT(CASE WHEN a.late_minutes > 0 THEN 1 END) AS late_count,
       SUM(a.late_minutes) AS total_late_minutes,
       ROUND(AVG(a.late_minutes), 1) AS avg_late
FROM employees e
JOIN attendances a ON e.id = a.employee_id
WHERE a.type = 'in'
AND a.attendance_date BETWEEN DATE_FORMAT(NOW(), '%Y-%m-01') AND CURDATE()
GROUP BY e.id, e.name
ORDER BY total_late_minutes DESC;
```

### تقارير الفروع

```sql
-- موظفو فرع معين
SELECT e.name, e.job_title, e.phone,
       CASE e.device_bind_mode
           WHEN 0 THEN 'حر'
           WHEN 1 THEN 'صارم'
           WHEN 2 THEN 'مراقبة'
       END AS bind_mode
FROM employees e
WHERE e.branch_id = ?
AND e.is_active = 1
AND e.deleted_at IS NULL;

-- مقارنة الحضور بين الفروع اليوم
SELECT b.name,
       COUNT(DISTINCT a.employee_id) AS present,
       (SELECT COUNT(*) FROM employees WHERE branch_id = b.id AND is_active = 1 AND deleted_at IS NULL) AS total
FROM branches b
LEFT JOIN employees e ON e.branch_id = b.id AND e.is_active = 1
LEFT JOIN attendances a ON a.employee_id = e.id AND a.attendance_date = CURDATE() AND a.type = 'in'
WHERE b.is_active = 1
GROUP BY b.id, b.name;
```

### صيانة قاعدة البيانات

```sql
-- حذف محاولات الدخول القديمة (أكثر من ساعة)
DELETE FROM login_attempts
WHERE attempted_at < NOW() - INTERVAL 60 MINUTE;

-- حجم الجداول
SELECT table_name,
       ROUND(data_length / 1024 / 1024, 2) AS data_mb,
       ROUND(index_length / 1024 / 1024, 2) AS index_mb,
       table_rows
FROM information_schema.tables
WHERE table_schema = DATABASE()
ORDER BY data_length DESC;
```
