## 1. رؤوس الأمان (Security Headers)

تُرسل مع **كل** استجابة HTTP:

| الرأس | القيمة | الحماية من |
|-------|--------|-----------|
| `X-Content-Type-Options` | `nosniff` | يمنع المتصفح من تخمين نوع الملف (MIME sniffing) |
| `X-Frame-Options` | `SAMEORIGIN` | يمنع عرض الموقع في iframe خارجي (Clickjacking) |
| `X-XSS-Protection` | `1; mode=block` | يفعّل حماية XSS في المتصفح |
| `Referrer-Policy` | `strict-origin-when-cross-origin` | يحد من معلومات Referrer المرسلة |
| `Permissions-Policy` | `geolocation=(self), camera=(), microphone=()` | يسمح بـ GPS فقط من نفس الموقع، يمنع الكاميرا والميكروفون |
| `Alt-Svc` | `clear` | يمنع توجيه البروتوكول |

## 2. حماية القوة الغاشمة (Brute Force Protection)

### كيف تعمل؟

```
محاولة دخول فاشلة #1 → يُسجّل في login_attempts
محاولة دخول فاشلة #2 → يُسجّل
محاولة دخول فاشلة #3 → يُسجّل
محاولة دخول فاشلة #4 → يُسجّل
محاولة دخول فاشلة #5 → يُسجّل
محاولة دخول #6       → ❌ IP مقفول لمدة 10 دقائق

بعد 10 دقائق → يُسمح بالمحاولة مرة أخرى
بعد 60 دقيقة → تُحذف السجلات القديمة تلقائياً
```

### الإعدادات

| المعامل | القيمة | الوصف |
|---------|--------|-------|
| `maxAttempts` | 5 | عدد المحاولات قبل القفل |
| `lockoutMinutes` | 10 | مدة القفل بالدقائق |
| `cleanupMinutes` | 60 | تنظيف السجلات بعد X دقيقة |

### الرسالة التي تظهر

```
"تم قفل الحساب مؤقتاً بسبب محاولات دخول كثيرة. حاول بعد 10 دقائق."
```

## 4. حماية CSRF

- كل نماذج HTML تحتوي `@csrf` (رمز Laravel CSRF)
- أي POST/PUT/DELETE بدون الرمز ← 419 Page Expired
- API endpoints تستخدم stateless auth (بدون session) ← لا تحتاج CSRF

## 5. إدارة الجلسات (Session Management)

### مهلة الخمول (Idle Timeout)

```
بدون "تذكرني":
  30 دقيقة بدون نشاط → تسجيل خروج تلقائي

مع "تذكرني":
  30 يوم → تسجيل خروج تلقائي
```

### تجديد الجلسة (Session Regeneration)

```
كل 5 دقائق → session()->regenerate()
يُغيّر معرّف الجلسة (Session ID) تلقائياً
يحمي ضد هجمات Session Hijacking و Session Fixation
```

### بيانات الجلسة المُخزّنة

```php
session([
    'admin_id'           => 1,
    'admin_name'         => 'مدير النظام',
    'session_created_at' => time(),
    'last_activity'      => time(),
]);
```

## 6. التحقق الجغرافي (Geofence Verification)

### كيف يعمل؟

```
1. المتصفح يُرسل GPS: latitude + longitude + accuracy
2. الخادم يحسب المسافة بين الموظف والفرع (صيغة Haversine)
3. إذا المسافة ≤ geofence_radius → ✅ مسموح
4. إذا المسافة > geofence_radius → ❌ مرفوض

لا يمكن للموظف التلاعب بـ GPS من الخادم
لكن يمكنه استخدام تطبيقات Fake GPS (ضعف معروف في جميع أنظمة GPS)
```

### الحماية الإضافية

- `location_accuracy` تُخزّن مع كل سجل (القيم العالية جداً تشير لـ fake GPS)
- `ip_address` تُخزّن للمراجعة
- `user_agent` يُخزّن لتحديد الجهاز

## 7. ربط الجهاز (Device Binding)

### الأوضاع الثلاثة

| الوضع | مستوى الحماية | السلوك |
|-------|-------------|--------|
| حر (0) | 🟢 منخفض | بدون ربط — أي جهاز مسموح |
| صارم (1) | 🔴 عالي | فقط الجهاز المسجّل — يُرفض الباقي + يُسجّل تلاعب |
| مراقبة (2) | 🟡 متوسط | أي جهاز مسموح لكن يُسجّل تنبيه |

### ماذا تعني "بصمة الجهاز"؟

بصمة الجهاز (Device Fingerprint) هي معرّف فريد يُولَّد من مواصفات المتصفح:
- دقة الشاشة
- المنطقة الزمنية
- اللغة
- WebGL renderer
- Canvas fingerprint
- الخطوط المتاحة
- وغيرها...

```
البصمة ≈ تجميع فريد لمواصفات الجهاز/المتصفح
مثل: "x7k2m9n4p8q1r3s5..."
```

## 8. سجل المراجعة (Audit Logging)

### ما الذي يُسجّل؟

| الإجراء | ماذا يُسجّل |
|---------|-----------|
| دخول المدير | IP, username, الوقت |
| خروج المدير | IP, الوقت |
| إنشاء/تعديل/حذف موظف | تفاصيل العملية + ID الموظف |
| حذف سجل حضور | ID السجل + ID الموظف |
| تحديث بلاغ | ID البلاغ + الحالة الجديدة |
| تحديث الإعدادات | تفاصيل التغيير |
| تغيير كلمة المرور | IP + الوقت (بدون كلمة المرور) |

### لماذا هذا مهم؟

- **المساءلة:** يمكن معرفة من قام بأي إجراء ومتى
- **التحقيق:** في حالة مشكلة يمكن مراجعة السجل
- **الامتثال:** متطلب قانوني في بعض الأنظمة

## 9. كشف التلاعب (Tampering Detection)

### أنواع التلاعب المكتشفة

| النوع | كيف يُكتشف | الخطورة |
|-------|-----------|---------|
| استخدام جهاز مختلف | بصمة الجهاز ≠ البصمة المسجّلة | عالية |

### ماذا يحدث عند اكتشاف تلاعب؟

```
1. يُنشأ سجل في جدول tampering_cases مع:
   - رقم الموظف
   - نوع التلاعب
   - تفاصيل (البصمة القديمة vs الجديدة + IP)
   - مستوى الخطورة
   - التاريخ

2. حسب وضع الربط:
   - صارم: يُرفض الطلب + يُسجّل التلاعب
   - مراقبة: يُقبل الطلب + يُسجّل تنبيه

3. المدير يرى التنبيهات في صفحة "كشف التلاعب"
```

## 10. حماية البيانات

### كلمات المرور

```
- تُشفّر بـ bcrypt (PHP Hash::make)
- bcrypt → أحادي الاتجاه (لا يمكن فك التشفير)
- كل تشفير ينتج قيمة مختلفة (salt عشوائي)
- Timing-safe comparison (يمنع هجمات التوقيت)
```

### رموز Token

```
- 64 حرف hex = 256 بت entropy
- يُولّد بـ random_bytes(32) → bin2hex
- احتمال التكرار: 1 من 2^256 (مستحيل عمليًا)
- يُخفى من استجابات JSON (hidden في Model)
```

### رموز PIN

```
- 4 أرقام = 10,000 احتمال
- محمي بـ rate limiting (10 محاولات/دقيقة)
- حتى مع rate limit: ~1000 دقيقة (16.7 ساعة) لتخمين كل الاحتمالات
- يُمكن إعادة توليده من لوحة المدير
```

### بيانات الموقع

```
- تُخزّن مع كل تسجيل حضور (latitude, longitude, accuracy)
- تُستخدم فقط للتحقق من الحضور
- لا تُشارك مع أطراف خارجية
- لا تُخزّن إلا لحظة التسجيل (لا تتبع مستمر)
```

## 11. حماية الحقن (Injection Protection)

### SQL Injection

```
✅ محمي: كل الاستعلامات عبر Laravel Eloquent ORM
✅ محمي: Parameterized queries (prepared statements)
✅ محمي: Input validation قبل أي query
✅ محمي: BackupService يستخدم PDO::quote() لهروب القيم عند إنشاء النسخ الاحتياطية
```

### XSS (Cross-Site Scripting)

```
✅ محمي: Blade {{ }} يُهرّب HTML تلقائياً
✅ محمي: Input validation بـ Laravel Validator
✅ محمي: Security Headers (X-XSS-Protection)
✅ محمي: Content-Security-Policy implicit
```

### CSRF (Cross-Site Request Forgery)

```
✅ محمي: @csrf في كل النماذج
✅ محمي: Laravel VerifyCsrfToken middleware
✅ محمي: رموز عشوائية لكل جلسة
✅ محمي: زر تسجيل الخروج يستخدم <form method="POST"> مع @csrf
```

## 13. المصادقة الثنائية (Two-Factor Authentication)

### كيف تعمل؟

```
1. المدير يُفعّل 2FA من الإعدادات
2. يُولَّد مفتاح سري (TOTP Secret) ويُعرض كـ QR Code
3. يُمسح QR Code بتطبيق مصادقة (Google Authenticator / Authy)
4. عند كل تسجيل دخول → يُطلب الرمز المتغير (6 أرقام) بالإضافة لكلمة المرور
```

### آلية العمل

| الخطوة | التفاصيل |
|--------|---------|
| **التفعيل** | `TwoFactorService::enable()` → يُولّد TOTP Secret + رموز استرداد |
| **التحقق** | `TwoFactorService::verify($code)` → يتحقق من الرمز عبر TOTP |
| **رموز الاسترداد** | 8 رموز مُشفّرة بـ `encrypt()` — تُستخدم مرة واحدة فقط |
| **التعطيل** | `TwoFactorService::disable()` → يُزيل السر ورموز الاسترداد |

### الحماية المضافة

```
بدون 2FA:
  اسم مستخدم + كلمة مرور = دخول ✅

مع 2FA:
  اسم مستخدم + كلمة مرور + رمز TOTP = دخول ✅
  اسم مستخدم + كلمة مرور فقط = ❌ يُطلب الرمز

فقدان الجهاز:
  اسم مستخدم + كلمة مرور + رمز استرداد = دخول ✅ (مرة واحدة)
```

### التخزين

| البيانات | العمود | التشفير |
|---------|--------|---------|
| مفتاح TOTP | `admins.two_factor_secret` | مُشفّر |
| رموز الاسترداد | `admins.two_factor_recovery_codes` | `encrypt()` — تشفير Laravel |
| حالة التفعيل | `admins.two_factor_enabled` | boolean (بدون تشفير) |

## 14. أمان Webhooks

### التوقيع الرقمي (HMAC-SHA256)

كل طلب Webhook يُوقّع رقمياً بالمفتاح السري الخاص بالـ webhook:

```
Signature = HMAC-SHA256(request_body_json, webhook_secret)
```

الهدف: يمنع أي طرف آخر من تزوير طلبات Webhook.

### كيف يتحقق المستقبِل؟

```php
// الطرف المستقبِل يتحقق من التوقيع
$expected = hash_hmac('sha256', $requestBody, $knownSecret);
$received = $request->header('X-Webhook-Signature');

if (!hash_equals($expected, $received)) {
    // ❌ التوقيع غير صالح — الطلب مرفوض
    abort(403);
}
```

### حماية إضافية

| الآلية | الوصف |
|-------|-------|
| **Timeout** | 10 ثوانٍ لكل طلب (يمنع DoS) |
| **تعطيل تلقائي** | بعد 10 فشل متتالي → الـ Webhook يُعطّل |
| **Secret فريد** | كل webhook له مفتاحه السري الخاص (64 حرف) |
| **سجل كامل** | كل محاولة إرسال تُسجّل في `webhook_logs` |

## 15. أمان Public API (Sanctum)

### المصادقة

- يتطلب Bearer Token صالح في كل الطلبات
- الرمز يُنشأ عبر `/api/tokens/create` بعد تقديم بيانات المدير
- كل رمز له صلاحيات محددة (`abilities`)
- يمكن إلغاء أي رمز عبر `/api/tokens/revoke`

### Rate Limiting

```
/api/tokens/create → 5 طلبات/دقيقة (حماية ضد تخمين كلمة المرور)
/api/v1/*          → Rate Limit قياسي
```

### أفضل الممارسات

1. **لا تُشارك الرمز** — عامله ككلمة مرور
2. **استخدم صلاحيات محدودة** — امنح `read` فقط إذا لم تحتج `write`
3. **ألغِ الرموز غير المُستخدمة** — لتقليل سطح الهجوم
4. **استخدم HTTPS فقط** — الرمز يُرسل في الرأس (لا تستخدم HTTP)

## 16. توصيات أمنية للمدير

### ✅ افعل

1. **غيّر كلمة المرور** بانتظام (كل 3 أشهر)
2. **استخدم كلمة مرور قوية** (12+ حرف، أرقام، رموز)
3. **فعّل الوضع الصارم** لربط الأجهزة (device_bind_mode = 1)
4. **راجع سجل التلاعب** أسبوعياً
5. **صغّر نطاق الجيوفينس** قدر الإمكان (200-300 متر)
6. **ضيّق النوافذ الزمنية** للحد من التسجيل المتأخر

### ❌ لا تفعل

1. لا تُشارك كلمة المرور مع أحد
2. لا تترك الحساب مفتوحاً على جهاز عام
3. لا تُعطّل ربط الأجهزة بدون سبب وجيه
4. لا تُوسّع الجيوفينس كثيراً (>1000 متر يُضعف الحماية)
5. لا تتجاهل تنبيهات التلاعب
