# 💡 اقتراحات تطوير النظام — رؤية مستقبلية

> **تاريخ الإعداد:** مارس 2026
> **الحالة:** اقتراحات للنقاش والتنفيذ التدريجي

---

## 🔴 الأولوية العاجلة

### 1. التحقق من دقة GPS
**المشكلة:** الموظف يمكنه إرسال إحداثيات بدقة مشكوك فيها أثناء استخدام تطبيقات Fake GPS.
**الحل المقترح:** رفض التسجيل إذا كانت دقة GPS أعلى من حد معين:
```php
if ($accuracy && $accuracy > 100) {
    return ['success' => false, 'message' => 'دقة GPS ضعيفة، حاول في مكان مكشوف'];
}
```
**الأثر:** حماية إضافية ضد تزوير الموقع.

---

### 2. Rate Limiting لكل موظف (Per-Employee)
**المشكلة:** الحد الحالي مبني على IP فقط — موظفون خلف NAT يشتركون في نفس IP.
**الحل المقترح:** إضافة rate limiting بالـ `unique_token` بالإضافة إلى الـ IP.
**الأثر:** حماية أدق ضد إساءة الاستخدام.

---

### 3. تشفير الجلسات
**المشكلة:** الجلسات مُخزّنة بدون تشفير إضافي (`encrypt => false` في `config/session.php`).
**الحل المقترح:** تفعيل `'encrypt' => true` في إعدادات الجلسة.
**الأثر:** حماية بيانات الجلسة من العبث.

---

## 🟡 الأولوية المتوسطة

### 4. إشعارات واتساب تلقائية
**الفكرة:** عند تسجيل الحضور أو الانصراف، يصل إشعار واتساب للموظف:
- "تم تسجيل حضورك الساعة 08:02 ✅"
- "تنبيه: لم يُسجَّل حضورك اليوم ⚠️"

**الموجود فعلاً:** `WebhookService` + `NotificationService` — يحتاج ربط مع WhatsApp API.

---

### 5. نظام إجازات متكامل من الموظف
**الوضع الحالي:** إضافة إجازات يدوية فقط من المدير.
**المطلوب:**
- طلب إجازة من واجهة الموظف عبر الهاتف
- موافقة/رفض من المدير مع إشعارات
- رصيد إجازات (سنوية/مرضية/طارئة)
- تقرير رصيد الإجازات المتبقية

---

### 6. تصدير تقارير PDF
**الوضع الحالي:** تصدير CSV + JSON فقط.
**المطلوب:**
- تصدير PDF مُنسَّق بشعار الشركة
- تقرير الرواتب (نموذج مبني على ساعات الحضور)
- تقرير العمل الإضافي شهرياً
- إرسال التقارير بالبريد الإلكتروني

**ملاحظة:** `AnalyticsService::getFinancialReport()` جاهز — يحتاج فقط عرض PDF.

---

### 7. PIN من 6 أرقام
**المشكلة:** PIN مكون من 4 أرقام (10,000 احتمال) — محمي بالـ rate limiting لكن ضعيف نسبياً.
**الحل المقترح:**
- رفع PIN إلى 6 أرقام (1,000,000 احتمال)
- إضافة قفل بعد 3 محاولات خاطئة لـ 15 دقيقة
- خيار بصمة الإصبع / Face ID (Web Authentication API)

---

### 8. إشعارات Push حقيقية
**الوضع الحالي:** إشعارات داخلية فقط (في الداشبورد).
**المطلوب:**
- Push Notifications بدون فتح التطبيق
- إشعار فوري عند:
  - حضور/انصراف الموظف
  - اكتشاف تلاعب
  - بلاغ سري جديد
  - إجازة معتمدة/مرفوضة

**التقنية:** Firebase Cloud Messaging (FCM) أو Web Push API.

---

### 9. نسخ احتياطي تلقائي مجدوَل
**الوضع الحالي:** نسخ يدوي من لوحة التحكم.
**الحل المقترح:**
```bash
# cron يعمل يومياً عند 2:00 صباحاً
0 2 * * * php /var/www/attendance/artisan backup:auto
```
- حفظ النسخة تلقائياً
- الاحتفاظ بآخر 30 نسخة
- إرسال إشعار بنجاح/فشل النسخ

## 🟢 الأولوية المنخفضة

### 10. تعدد المشرفين مع صلاحيات دقيقة
**المشكلة:** مشرف واحد فقط بصلاحيات كاملة.
**الحل المقترح:**
```
أدوار مقترحة:
  - super_admin: كل الصلاحيات
  - branch_manager: صلاحيات فرعه فقط
  - hr_viewer: يشاهد التقارير فقط بدون تعديل
  - auditor: يشاهد سجل التدقيق فقط
```

---

### 11. التكامل مع أنظمة الرواتب
**الفكرة:** تصدير بيانات الحضور بصيغ مدعومة من:
- صيغة Excel/CSV مخصصة
- نظام JSON API للتكامل المباشر

**الموجود:** `PublicApiController` + Sanctum Auth جاهزان — يحتاج فقط تخصيص الصيغ.

---

### 12. إشعارات بريد إلكتروني
**الفكرة:**
- بريد يومي للمشرف بملخص حضور الفروع
- تنبيه فوري عند تسجيل مشبوه
- تقرير أسبوعي PDF مرفق

---

### 13. واجهة متعددة اللغات
**الوضع الحالي:** عربي فقط.
**الحل المقترح:** نظام ترجمة Laravel (`lang/ar/` + `lang/en/` — البنية موجودة فعلاً).

---

### 14. تحسين PWA
**الوضع الحالي:** واجهة ويب متجاوبة.
**المطلوب:**
- Service Worker + manifest.json
- وضع offline مع مزامنة عند الاتصال
- Widget للشاشة الرئيسية

---

### 15. نظام إدارة المشاريع المتقدم
**الموجود:** نماذج `Project`, `Task`, `TimeEntry` مُعرّفة.
**المطلوب:**
- واجهة إدارة مشاريع + مهام
- تتبع وقت العمل على المهام
- تقارير إنتاجية
