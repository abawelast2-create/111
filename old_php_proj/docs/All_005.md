# نظام الحضور والانصراف — نظرة عامة

## ما هو النظام؟

نظام ويب متكامل لتسجيل حضور وانصراف الموظفين عبر الهاتف الذكي، يعتمد على:
- **الموقع الجغرافي (GPS)** — لا يمكن التسجيل إلا من داخل نطاق العمل
- **دعم الفروع المتعددة** — كل فرع له إحداثياته ونصف قطره الخاص
- **ورديات متعددة** — حتى 3 ورديات لكل فرع مع كشف تلقائي للوردية النشطة
- **رابط فريد لكل موظف** — بدلاً من كلمات المرور
- **نظام PIN** — رقم سري فريد (4 أرقام) لكل موظف
- **باركود QR** — توليد QR Code لبوابة الموظف مع شعار التطبيق
- **ربط الجهاز (3 أوضاع)** — حر / صارم (يمنع) / مراقبة صامتة (يسجل التلاعب بصمت)
- **تتبع ملكية الأجهزة** — النظام يعرف تلقائياً لمن ينتمي كل جهاز بناءً على الاستخدام
- **كشف التلاعب** — تسجيل بالنيابة، أجهزة مختلفة، مع سجل كامل
- **بلاغات سرية** — زر عائم للموظفين لإرسال بلاغات مع نص/صور/صوت (مع مؤثرات صوتية)
- **خلفية صور متحركة** — عرض شرائح صور كخلفية لصفحة الحضور
- **واجهة عربية** — تصميم برتقالي عصري يدعم RTL بالكامل
- **خريطة تفاعلية** — خريطة Leaflet تُظهر مركز العمل ونطاقه في واجهة الموظف
- **صوت رادار** — صوت سونار عند فتح الصفحة (Web Audio API)
- **شعار مخصص** — الشعار في كل مكان مع أنيميشن متعددة
- **تنبيه GPS مستمر** — نافذة منبثقة لتفعيل الموقع
- **PWA** — تطبيق ويب تقدمي مع Service Worker
- **Rate Limiting** — حماية من الطلبات المتكررة على API

---

## المصطلحات المهمة

| المصطلح | التعريف |
|---------|---------|
| **PIN** | رقم سري فريد من 4 أرقام يُعطى لكل موظف — يُستخدم لدخول بوابة الموظف |
| **Token** | نص عشوائي (64 حرف hex) يُمثّل رابط الموظف الفريد — بديل عن كلمة المرور |
| **Geofence** | السياج الجغرافي — نطاق دائري حول موقع العمل يُحدَّد بالمتر |
| **نافذة زمنية** | الفترة المسموح فيها بالتسجيل — من `shift_start - 60 دقيقة` حتى `shift_end` |
| **ربط الجهاز** | ربط بصمة جهاز الموظف بحسابه — 3 أوضاع: حر / صارم / مراقبة |
| **بصمة الجهاز (Fingerprint)** | قيمة SHA-256 تُحسب من خصائص المتصفح والجهاز — تُستخدم لتمييز الأجهزة |
| **الوردية (Shift)** | فترة عمل محددة ببداية ونهاية — كل فرع يدعم حتى 3 ورديات |
| **العمل الإضافي (Overtime)** | ساعات بعد الانصراف — يبدأ بعد فترة محددة وله حد أدنى |
| **التأخر (Late)** | الفرق بين `shift_start` ووقت تسجيل الحضور الفعلي — يُحسب بالدقائق |
| **Soft Delete** | أرشفة الموظف بدون حذف بياناته — يُستعاد بإزالة `deleted_at` |
| **Haversine** | صيغة رياضية لحساب المسافة بين نقطتين على سطح الكرة الأرضية |

1. يضيف الموظف من لوحة الإدارة

3. يصل للمكتب → يضغط "تسجيل الحضور"
4. في نهاية الدوام → يضغط "تسجيل الانصراف"
5. يمكنه إرسال بلاغ سري عبر الزر العائم (🛡️)
```

### أولوية الإعدادات (Settings Priority)

النظام يتبع أولوية واضحة عند جلب الإعدادات:

| الأولوية | المصدر | مثال |
|----------|--------|------|
| 1 (الأعلى) | إعدادات الفرع (`branches`) | `geofence_radius`, `allow_overtime`, أوقات الورديات |
| 2 (الافتراضي) | الإعدادات العامة (`settings`) | `work_latitude`, `geofence_radius`, أوقات الدوام |

**القاعدة:** إذا الموظف مرتبط بفرع (`branch_id IS NOT NULL`) → يُستخدم إعداد الفرع.  
إذا لم يكن مرتبطاً بفرع → تُستخدم الإعدادات العامة.

```php
// مثال في isWithinGeofence():
if ($branchId) {
    $branch = getBranch($branchId);
    $lat = $branch['latitude'];
    $radius = $branch['geofence_radius'];
} else {
    $lat = getSystemSetting('work_latitude');
    $radius = getSystemSetting('geofence_radius');
}
```

## استعلامات شائعة مرجعية

```sql
-- إحصائيات اليوم
SELECT
  (SELECT COUNT(DISTINCT employee_id) FROM attendances WHERE DATE(timestamp) = CURDATE() AND type = 'in') AS checked_in,
  (SELECT COUNT(DISTINCT employee_id) FROM attendances WHERE DATE(timestamp) = CURDATE() AND type = 'out') AS checked_out,
  (SELECT COUNT(*) FROM employees WHERE is_active = 1) AS total;

-- موظفي فرع معين
SELECT e.*, b.name AS branch_name FROM employees e
LEFT JOIN branches b ON e.branch_id = b.id
WHERE b.id = ?;

-- حضور موظف معين اليوم
SELECT type, timestamp FROM attendances
WHERE employee_id = ? AND DATE(timestamp) = CURDATE()
ORDER BY timestamp ASC;

-- الغائبون اليوم
SELECT e.name, e.job_title, b.name AS branch_name FROM employees e
LEFT JOIN branches b ON e.branch_id = b.id
WHERE e.is_active = 1
AND e.id NOT IN (
  SELECT DISTINCT employee_id FROM attendances WHERE DATE(timestamp) = CURDATE()
);

-- تقرير موظف لفترة محددة
SELECT a.type, a.timestamp, a.latitude, a.longitude
FROM attendances a
WHERE a.employee_id = ? AND a.timestamp BETWEEN ? AND ?
ORDER BY a.timestamp;

## 4. التحقق من بصمة الجهاز

**`POST /api/verify-device.php`**

### الطلب

```json
{
  "token": "56f3a733e65444450bc8...",
  "fingerprint": "a3f9c2d1e8b4..."
}
```

| الحقل | الوصف |
|-------|-------|
| `token` | الرمز الفريد للموظف |
| `fingerprint` | SHA-256 لبصمة الجهاز (64 حرف hex) |

### أوضاع ربط الجهاز (`device_bind_mode`)

| القيمة | الوضع | السلوك |
|--------|-------|--------|
| 0 | حر | الرابط يعمل من أي جهاز بدون قيود |
| 1 | صارم | الرابط مربوط بجهاز واحد — أي جهاز آخر يُحظر |
| 2 | مراقبة صامتة | يسمح بالدخول من أي جهاز لكن يُسجّل التلاعب بصمت |

### تتبع الأجهزة (`known_devices`)

عند كل دخول، يتم تسجيل/تحديث الجهاز في جدول `known_devices`:
- إذا كان الجهاز موجوداً: يزيد `usage_count` ويُحدّث `last_used_at`
- إذا كان جديداً: يُنشأ سطر جديد بـ `usage_count = 1`
- **مالك الجهاز:** الموظف صاحب أعلى `usage_count` لنفس البصمة

### الاستجابات المحتملة

**الوضع الحر (`device_bind_mode = 0`) — أول دخول:**
```json
{
  "success": true,
  "first_time": true,
  "auto_bound": false
}
```
> يدخل بدون ربط. لربط الجهاز، يُفعّل المشرف وضع الربط يدوياً.

**أول دخول (الربط مفعّل يدوياً من المشرف):**
```json
{
  "success": true,
  "first_time": true,
  "auto_bound": true
}
```
> تم ربط الجهاز وتحويل `device_bind_mode` إلى الوضع المختار.

**جهاز مطابق (أي وضع):**
```json
{
  "success": true,
  "first_time": false
}
```

**جهاز مختلف — الوضع الصارم (`device_bind_mode = 1`):**
```json
{
  "success": false,
  "locked": true,
  "message": "هذا الرابط مرتبط بجهاز آخر. تواصل مع المشرف لإعادة تعيين الجهاز."
}
```
> يُحظر الدخول ويُعرض شاشة قفل.

**جهاز مختلف — المراقبة الصامتة (`device_bind_mode = 2`):**
```json
{
  "success": true,
  "first_time": false
}
```
> يُسمح بالدخول لكن يُسجّل حالة تلاعب `different_device` في جدول `tampering_cases` مع تفاصيل البصمة وIP ومالك الجهاز.

## حساب بصمة الجهاز (JavaScript)

```javascript
async function getFingerprint() {
  const parts = [
    navigator.userAgent,
    screen.width + 'x' + screen.height + 'x' + screen.colorDepth,
    Intl.DateTimeFormat().resolvedOptions().timeZone,
    navigator.language,
    navigator.platform || ''
  ];
  // Canvas fingerprint — بصمة رسومية فريدة
  const c = document.createElement('canvas');
  const x = c.getContext('2d');
  x.font = '14px Arial'; x.fillText('fp', 2, 2);
  parts.push(c.toDataURL().slice(-50));

  const raw = parts.join('|');
  const buf = await crypto.subtle.digest('SHA-256', new TextEncoder().encode(raw));
  return Array.from(new Uint8Array(buf))
    .map(b => b.toString(16).padStart(2, '0')).join('');
}
```

### 4. `04-الأمان.md`

```markdown
# الأمان — التوثيق الكامل

## طبقات الحماية المنفّذة

---

### 1. نظام المصادقة

#### مشرف النظام
- كلمة مرور مشفّرة بـ **bcrypt** (`password_hash` + `password_verify`)
- **CSRF Token** على جميع نماذج POST — يمنع هجمات Cross-Site Request Forgery
- تجديد معرّف الجلسة عند كل تسجيل دخول (`session_regenerate_id(true)`)
- الجلسة تنتهي بإغلاق المتصفح (`session.cookie_lifetime = 0`)
- `session.cookie_httponly = 1` ← JavaScript لا يستطيع قراءة الكوكي
- `session.cookie_samesite = Strict` ← حماية إضافية من CSRF

#### الموظف
- لا يوجد كلمة مرور — الهوية عبر **unique_token** (64 حرف عشوائي)
- التوكن يُنتج بـ `bin2hex(random_bytes(32))` ← آمن كريبتوغرافياً
- لا يمكن تخمينه: احتمال 1 من 2^256

---

### 2. قفل الجهاز (Device Fingerprinting) — 3 أوضاع

#### أوضاع الربط (`device_bind_mode`)

| الوضع | القيمة | السلوك |
|------|--------|--------|
| حر | 0 | الرابط يعمل من أي جهاز بدون قيود |
| صارم | 1 | يربط الجهاز ويمنع أي جهاز مختلف (شاشة قفل 🔒) |
| مراقبة صامتة | 2 | يسمح بالدخول لكن يُسجّل التلاعب في جدول `tampering_cases` |

#### آلية العمل
```
الدخول الأول (وضع حر):
  JS يحسب بصمة SHA-256 ← ترسل للـ API
  ← لا توجد بصمة محفوظة + وضع حر
  ← الموظف يدخل بدون ربط ✅

تفعيل الربط الصارم من المشرف:
  المشرف يضغط "🔒 ربط صارم" ← device_bind_mode = 1
  ← الموظف يفتح الرابط ← الجهاز يُربط تلقائياً ✅
  ← أي جهاز آخر ← شاشة قفل ❌

تفعيل المراقبة الصامتة من المشرف:
  المشرف يضغط "👁️ ربط مراقبة" ← device_bind_mode = 2
  ← الموظف يفتح الرابط ← الجهاز يُربط تلقائياً ✅
  ← جهاز مختلف ← يدخل لكن يُسجّل التلاعب بصمت 🕵️
```

#### تتبع الأجهزة (known_devices)
- كل دخول يُسجّل في `known_devices` (INSERT ON DUPLICATE KEY UPDATE)
- يتتبع `usage_count` لكل جهاز+موظف
- **مالك الجهاز:** الموظف صاحب أعلى `usage_count` لنفس البصمة
- يظهر اسم مالك الجهاز (📱) في عمود الجهاز بلوحة الإدارة

#### مكونات البصمة
1. `userAgent` — نظام التشغيل والمتصفح
2. `screen.width × height × colorDepth` — مواصفات الشاشة
3. `Intl timezone` — المنطقة الزمنية
4. `navigator.language` — لغة الجهاز
5. `canvas fingerprint` — بصمة رسومية فريدة للـ GPU

#### مقاومة التزوير
- `hash_equals()` في PHP ← يمنع **Timing Attacks**
- البصمة SHA-256 لا تُعكس ← المهاجم لا يستطيع استخراج البيانات الأصلية

---

### 3. كشف التلاعب (Tampering Detection)

يُسجّل حالات التلاعب تلقائياً في جدول `tampering_cases`:

| نوع التلاعب | الوصف | الخطورة |
|------------|-------|--------|
| `different_device` | دخول من جهاز مختلف عن المربوط | متوسطة |
| `proxy_checkin` | تسجيل بالنيابة (مستقبلي) | عالية |
| `location_spoof` | تزوير الموقع (مستقبلي) | عالية |

**التفاصيل المُسجّلة** (`details_json`):

### 5. الحماية من الحقن (Injection)

#### SQL Injection
- جميع استعلامات DB تستخدم **Prepared Statements + PDO**
- `PDO::ATTR_EMULATE_PREPARES = false` ← Prepared Statements حقيقية

```php
// ✅ آمن دائماً
$stmt = db()->prepare("SELECT * FROM employees WHERE unique_token = ?");
$stmt->execute([$token]);
```

#### XSS (Cross-Site Scripting)
- دالة `sanitize()` على جميع المدخلات: `htmlspecialchars + strip_tags + trim`
- `htmlspecialchars()` على كل إخراج ديناميكي في HTML

                              ┌─────────────────────┘

## التخزين المؤقت (Caching)

### نمط الـ Static Cache في الإعدادات

```php
// includes/functions.php
function loadAllSettings(bool $refresh = false): array {
    static $cache = null;       // ← تُخزَّن في ذاكرة PHP لمدة الطلب
    if ($cache !== null && !$refresh) {
        return $cache;          // ← يُعاد الـ cache بدون استعلام DB
    }
    $cache = fetchAllFromDB();  // ← استعلام واحد: SELECT * FROM settings
    return $cache;
}
```

**كيف يعمل:**
```
الطلب الأول في نفس HTTP request:
  getSystemSetting('work_latitude')
  → loadAllSettings() → SELECT * FROM settings → $cache = [...]
  → يُعيد القيمة من $cache

الطلب الثاني في نفس HTTP request:
  getSystemSetting('geofence_radius')
  → loadAllSettings() → $cache موجود
  → يُعيد القيمة مباشرة بدون DB query
```

**التحديث:**
```php
setSystemSetting('key', 'value');
// → UPDATE settings SET ... WHERE setting_key = ?
// → loadAllSettings(refresh: true)  ← يُبطل الـ cache
```

> **ملاحظة:** هذا "Per-Request Cache" — يعيش فقط خلال طلب HTTP واحد. لا يُخزَّن بين الطلبات (لا Redis/Memcached).
```

## المرحلة 1: تسجيل الحضور (Check-In)

### 1.1 — الموظف يفتح الرابط

الموظف يفتح الرابط المرسل عبر واتساب:

```
GET sarh.io/xml/employee/attendance.php?token=a7b3c9d1e5f2...
```

**ما يحدث في الكود:**
```php
$token = $_GET['token'];
$employee = getEmployeeByToken($token);
// → SELECT * FROM employees WHERE unique_token = ? AND is_active = 1 AND deleted_at IS NULL
```

### 1.2 — التحقق من الجهاز (verify-device)

JavaScript يحسب بصمة الجهاز (SHA-256) ويُرسلها:

```
POST /api/verify-device.php
Body: { "token": "a7b3c9d1...", "fingerprint": "x7k2m9..." }
```

**حسب `device_bind_mode`:**

| الحالة | الوضع | النتيجة | تغييرات DB |
|--------|-------|---------|-----------|
| أول دخول + حر (0) | حر | ✅ مسموح | `known_devices` += سجل جديد |
| أول دخول + صارم (1) | صارم | ✅ يُربط | `employees.device_fingerprint` + `known_devices` |
| نفس الجهاز | أي وضع | ✅ مسموح | `known_devices.usage_count` += 1 |
| جهاز مختلف + صارم | صارم | ❌ مرفوض | `tampering_cases` سجل جديد |
| جهاز مختلف + مراقبة | مراقبة | ✅ يدخل | `tampering_cases` سجل جديد (بصمت) |

### 1.3 — صفحة الحضور تُحمّل

```
GET /employee/attendance.php?token=a7b3c9d1...
```

**استعلامات قراءة فقط:**
```sql
SELECT * FROM employees WHERE unique_token = 'a7b3c9d1...' AND is_active = 1;
SELECT * FROM branches WHERE id = 1 AND is_active = 1;
SELECT * FROM branch_shifts WHERE branch_id = 1 AND is_active = 1 ORDER BY shift_number;
SELECT * FROM attendances WHERE employee_id = 1 AND attendance_date = '2026-03-15' ORDER BY timestamp;
```

**JavaScript يبدأ:**
- رادار GPS + أنيميشن
- صوت سونار
- خلفية صور متحركة
- تتبع الموقع الجغرافي

### 1.4 — المتصفح يحدد الموقع

       = ✅ داخل النطاق
```

### 1.5 — الموظف يضغط "تسجيل الحضور"

**طلب API:**
```

**ما يحدث في الكود خطوة بخطوة:**

```

   → الوردية 1 (08:00-16:00) ← النافذة: 07:00 - 16:00
5. الوقت الحالي 08:15 → ✅ ضمن النافذة
6. isWithinGeofence(24.57238, 46.60248, 1)
   → المسافة 10 متر ≤ 200 متر → ✅ داخل النطاق
7. hasRecentAttendance(1, 'in', 5)
   → لا يوجد تسجيل خلال 5 دقائق → ✅
8. recordAttendance(1, 'in', 24.57238, 46.60248, 15.0)
   → يحسب التأخر: work_start = 08:00, الآن = 08:15 → 15 دقيقة

## المرحلة 7: دخول المشرف وعرض التقارير

### 7.1 — تسجيل دخول المشرف

```
POST /admin/login.php
Body: { username: "admin", password: "..." }
```

**الخطوات:**

1. **حماية القوة الغاشمة:**
```
يفحص login_attempts لنفس IP خلال آخر 10 دقائق
إذا العدد ≥ الحد → مقفول
```

2. **التحقق:**
```php
$admin = SELECT * FROM admins WHERE username = ?;
password_verify($password, $admin['password_hash'])
```

3. **عند النجاح:**
```
┌ admins (تحديث) ─────────────────────────┐
│ last_login: 2026-03-15 14:00:00          │
└──────────────────────────────────────────┘

┌ audit_log (سجل جديد) ───────────────────┐
│ action: "login"                           │
│ details: "تسجيل دخول: admin"            │
│ admin_id: 1                               │
│ ip_address: "188.XX.XX.XX"               │
└───────────────────────────────────────────┘

+ session_regenerate_id(true)
+ $_SESSION['admin_id'] = 1
```

4. **عند الفشل:**
```
┌ login_attempts (سجل جديد) ───────────────┐
│ ip_address: "188.XX.XX.XX"               │
│ username: "admin"                         │
│ attempted_at: 2026-03-15 14:00:00        │
│ success: 0                                │
└───────────────────────────────────────────┘
```

### 7.2 — لوحة القيادة (Dashboard)

```
GET /admin/dashboard.php
```

**الاستعلامات التي تُنفّذ:**
```sql
-- 1. إحصائيات اليوم
SELECT
  COUNT(DISTINCT CASE WHEN type='in' THEN employee_id END) as checked_in,
  COUNT(DISTINCT CASE WHEN type='out' THEN employee_id END) as checked_out,
  (SELECT COUNT(*) FROM employees WHERE is_active=1 AND deleted_at IS NULL) as total
FROM attendances WHERE attendance_date = CURDATE();

-- 2. عدد الفروع النشطة
SELECT COUNT(*) FROM branches WHERE is_active = 1;

-- 3. آخر 10 تسجيلات
SELECT a.*, e.name, e.job_title, b.name as branch_name
FROM attendances a
JOIN employees e ON a.employee_id = e.id
LEFT JOIN branches b ON e.branch_id = b.id
ORDER BY a.timestamp DESC LIMIT 10;

-- 4. الغائبون
SELECT e.name, e.job_title, b.name as branch_name
FROM employees e
LEFT JOIN branches b ON e.branch_id = b.id
WHERE e.is_active = 1 AND e.deleted_at IS NULL
AND e.id NOT IN (
    SELECT DISTINCT employee_id FROM attendances
    WHERE attendance_date = CURDATE() AND type = 'in'
);
```

**تتحدث البيانات تلقائياً عبر AJAX (realtime-dashboard.php).**

### 7.3 — تقرير التأخر

```
GET /admin/late-report.php?from=2026-03-01&to=2026-03-15
```

```sql
SELECT a.*, e.name, b.name as branch_name
FROM attendances a
JOIN employees e ON a.employee_id = e.id
LEFT JOIN branches b ON e.branch_id = b.id
WHERE a.type = 'in' AND a.late_minutes > 0
AND a.attendance_date BETWEEN '2026-03-01' AND '2026-03-15'
ORDER BY a.timestamp DESC;
```

**المعالجة:**
```
لكل موظف:
  عدد مرات التأخر = count(سجلات التأخر)
  إجمالي الدقائق = sum(late_minutes)
  المعدل = إجمالي / عدد المرات
  التقييم:
    ≤ 5 دقائق متوسط ← "ممتاز" 🟢
    ≤ 15 دقيقة متوسط ← "مقبول" 🟡
    > 15 دقيقة متوسط ← "ضعيف" 🔴
```

### 7.4 — تصدير البيانات (CSV)

```
GET /api/export.php?format=csv&date_from=2026-03-01&date_to=2026-03-15
```

**ينتج ملف CSV بهذا الشكل:**
```csv

أحمد محمد,out,2026-03-15,16:05,0,الفرع الرئيسي
```

> يُضاف UTF-8 BOM للتوافق مع Excel. حماية من CSV Formula Injection.

  - `known_devices.employee_id → employees.id (ON DELETE CASCADE)`
  - `leaves` يرتبط بـ `employees` و `admins`
- **الحذف الناعم (Soft Delete):** عمود `deleted_at` في `employees` — يحافظ على البيانات التاريخية.
- **ENUM للبيانات المحدودة:** `attendances.type` محصور في `('in', 'out', 'overtime')`, `leaves.status` في `('pending', 'approved', 'rejected')`.
- **أعمدة التدقيق:** `created_at` موجود في جميع الجداول, `ip_address` و `user_agent` في `attendances`.

#### مستوى التطبيع (Normalization)
- **3NF متحقق:** لا توجد تكرارات واضحة. الإعدادات في جدول Settings منفصل (Key-Value).
- **ملاحظة:** جدول `branches` يحوي أعمدة مواعيد (12 عموداً زمنياً) — يمكن فصلها إلى جدول `branch_schedules` إذا كانت هناك ورديات متعددة لكل فرع.

#### نقاط الضعف
| المشكلة | الخطورة | التوصية |
|---------|---------|---------|
| لا يوجد جدول `tampering_cases` ضمن `install.php` بشكل صريح | ⚠️ متوسطة | `verify-device.php` ينشئه ديناميكياً بـ `CREATE TABLE IF NOT EXISTS` — يُفضّل إنشاؤه في المثبّت |
| `secret_reports` غير موجود في `install.php` | ⚠️ متوسطة | يجب إنشاؤه في سكربت التثبيت |
| لا يوجد حد أقصى لحجم `audit_log` | 🟡 منخفضة | إضافة مهمة تنظيف دورية أو تقسيم (partitioning) |
| `employees.pin` بحد أقصى 10 أحرف — TEXT | 🟡 منخفضة | كافٍ كرقم تعريفي داخلي |
| `attendances.user_agent` من نوع `TEXT` | 🟡 منخفضة | يمكن تحويله إلى `VARCHAR(512)` لتوفير المساحة |

## 7. الميزات الرئيسية

### 7.1 نظام الجيوفينس
- **صيغة Haversine:** حساب دقيق للمسافة على سطح الكرة الأرضية — يتم على الخادم (لا يمكن تزويره من المتصفح).
- **دعم الفروع المتعددة:** كل فرع بإحداثيات ونصف قطر مستقلين.
- **تدريج ذكي:** إذا لم يُحدد فرع، يُستخدم الإعدادات العامة. إذا لم يُحدد موقع أصلاً، يُسمح.

### 7.2 بصمة الجهاز (Device Fingerprinting)
- **5 مكونات:** UserAgent + أبعاد الشاشة + المنطقة الزمنية + اللغة + Canvas fingerprint.
- **3 أوضاع ربط:** حر (0) / صارم (1) / مراقبة صامتة (2) — تصميم مرن وذكي.
- **تتبع الملكية:** جدول `known_devices` يتتبع عدد مرات الاستخدام ويحدد المالك.
- **كشف التسجيل بالنيابة:** إذا استُخدم نفس الجهاز لموظفين مختلفين، يُسجل `proxy_checkin`.
- **حماية من Timing Attacks:** استخدام `hash_equals()` بدلاً من `===`.

### 7.3 الروابط الفريدة
- 64 حرف hex (256-bit entropy) — مستحيل التخمين.
- قابلة للتجديد فردياً أو جماعياً.
- مُرسلة عبر واتساب بنقرة واحدة.

### 7.4 الدوام الإضافي
- تفعيل/تعطيل عام أو لكل فرع.
- ضوابط: يبدأ بعد X دقيقة من الانصراف، حد أدنى للمدة.
- يتطلب تسجيل انصراف أولاً.

### 7.5 نظام الورديات المزدوجة
- وردية صباحية + مسائية مع كشف تلقائي.
- دعم تجاوز منتصف الليل (midnight crossing).
- الأنصراف التلقائي عبر Cron (`auto-checkout.php`).

### 7.6 البلاغات السرية
- **نص + صور + تسجيل صوتي** مع مؤثرات تغيير الصوت.
- 4 مؤثرات: عادي / عميق / حاد / روبوت — لحماية هوية المُبلِّغ.
- أنواع البلاغات: عام / مخالفة / تحرش / سرقة / سلامة / أخرى.

### 7.7 الانصراف التلقائي (Cron)
- يفحص كل دقيقة.
- يدعم الورديتين ويكشف الوردية عبر وقت الدخول.
- يدعم تجاوز منتصف الليل.
- حماية CLI + secret key.

### 7.8 التحديث اللحظي (Real-time)
- **لوحة التحكم:** AJAX polling يحدّث الإحصائيات وآخر التسجيلات لحظياً.
- **صفحة الحضور:** بيانات الحضور تتحدث بدون refresh.
- **مؤشر حي (Live Indicator):** نقطة انميشن خضراء تشير لاتصال نشط.

## 8. نقاط القوة

### 8.1 التصميم والمعمارية
1. **خفة التشغيل:** بدون أطر عمل ثقيلة — كل طلب PHP ينتهي في أجزاء من الثانية.
2. **هيكل ملفات واضح:** فصل نظيف بين API وAdmin وEmployee وIncludes.
3. **توثيق استثنائي:** 8 ملفات توثيق عربية شاملة — نادر في المشاريع بهذا الحجم.
4. **Singleton Pattern:** اتصال PDO واحد لكل طلب — يمنع تسرب الاتصالات.
5. **Settings Cache:** `loadAllSettings()` يحمّل كل الإعدادات دفعة واحدة مع static caching.

### 8.2 الأمان
6. **طبقات أمنية متعددة:** bcrypt + CSRF (One-Time) + Rate Limiting + Brute Force Protection + Device Binding + Geofencing.
7. **المراقبة الصامتة (وضع 2):** ابتكار ذكي يسمح بكشف التلاعب دون إنذار الموظف.
8. **سجل تدقيق شامل:** كل عملية إدارية تُسجّل مع IP ورقم المشرف.
9. **IP Spoofing Protection:** `getClientIP()` يثق فقط بـ proxies محددة.
10. **حماية تسرب معلومات DB:** رسالة خطأ عامة عند فشل الاتصال (`display_errors=0`).

### 8.3 تجربة المستخدم
11. **صفر تثبيت:** الموظف يفتح رابطاً فقط — لا تطبيق، لا تسجيل دخول.
12. **تصميم Mobile-First:** واجهة الموظف مصممة للهاتف حصراً.
13. **الرادار التفاعلي:** تجربة بصرية فريدة مع Canvas Animation.
14. **دعم عربي كامل:** RTL + خط Tajawal + أيقونات SVG مدمجة.
15. **PWA:** قابل للتثبيت كتطبيق على الشاشة الرئيسية.

### 8.4 الذكاء البرمجي
16. **كشف الوردية التلقائي:** يحدد الوردية بناءً على الوقت الحالي أو وقت الدخول.
17. **دعم تجاوز منتصف الليل:** منطق متين في check-in/check-out/auto-checkout.
18. **الإنشاء التلقائي للجداول:** `CREATE TABLE IF NOT EXISTS` في verify-device — يعمل حتى بدون تشغيل المثبت.
19. **Auto-detect SITE_URL:** يكتشف البروتوكول والمسار تلقائياً — لا حاجة لتعديل يدوي.
20. **تغيير الصوت بـ Web Audio API:** تنفيذ كامل للتحويل الصوتي على المتصفح بدون خادم.
