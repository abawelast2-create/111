## includes/config.php — الإعدادات الأساسية

### الثوابت المعرّفة

| الثابت | القيمة الافتراضية | الوصف |
|--------|------------------|-------|
| `DB_HOST` | `localhost` | عنوان خادم MySQL |
| `DB_USER` | `root` | مستخدم قاعدة البيانات |
| `DB_PASS` | `''` | كلمة المرور |
| `DB_NAME` | `attendance_system` | اسم قاعدة البيانات |
| `DB_CHARSET` | `utf8mb4` | ترميز الاتصال |
| `SITE_URL` | *(يُكشف تلقائياً)* | رابط الموقع (Auto-detect من بروتوكول + host + path) |
| `SITE_NAME` | `نظام الحضور والانصراف` | اسم الموقع |

> **ملاحظة:** يمكن تخصيص الثوابت عبر ملف `.env` في جذر المشروع (إن وُجد).

## أنماط المصادقة (Authentication Patterns)

### 1. مصادقة المشرف (Session-Based)

```
┌──────────┐     POST /admin/login.php      ┌──────────────┐
│  المشرف  │ ────────────────────────────→  │  auth.php    │
│ (متصفح)  │  username + password           │ adminLogin() │
└──────────┘                                └──────┬───────┘
                                                   │
                    ┌──────────────────────────────┘
                    │
            password_verify($pw, $hash)
                    │
              ┌─────┴─────┐
              │           │
           ✅ نجح      ❌ فشل
              │           │
     session_start()    login_attempts++
     $_SESSION['admin_id']=1     │
     last_login = NOW()     if attempts ≥ limit:
     audit_log++                 LOCK 10 min
              │
              ▼
     كل صفحة admin/:
     requireAdminLogin()
     → يفحص $_SESSION['admin_id']
     → إذا لا يوجد → redirect login.php
```

### 2. مصادقة الموظف (Token-Based)

```
┌──────────┐     GET ?token=a7b3c9d1...     ┌──────────────┐
│  الموظف  │ ────────────────────────────→  │ functions.php│
│ (هاتف)   │                                │getEmployeeBy │
└──────────┘                                │   Token()    │
                                            └──────┬───────┘
                                                   │
                    SELECT * FROM employees
                    WHERE unique_token = ?
                    AND is_active = 1
                    AND deleted_at IS NULL
                                                   │
                              ┌─────────────────────┘
                              │
                   ┌──────────┴──────────┐
                   │                     │
                ✅ وُجد              ❌ لم يُوجد
                   │                     │
            verify-device.php       403 Forbidden
            → فحص بصمة الجهاز
                   │
            attendance.php يُحمّل
            → GPS + رادار + أزرار
```

### 3. مصادقة PIN (بوابة الموظف)

```
POST /api/auth-pin.php → { pin: "3847" }
→ getEmployeeByPin($pin)
→ إذا وُجد + is_active → ✅ بيانات الموظف
→ إذا لم يُوجد → ❌ PIN غير صحيح
```

## نمط التحقق (Validation Patterns)

### قواعد التحقق لكل نقطة نهاية

**check-in.php / check-out.php:**
```
token      → string, 64 حرف hex (مطلوب)
latitude   → numeric, -90 إلى 90 (مطلوب)
longitude  → numeric, -180 إلى 180 (مطلوب)
accuracy   → numeric, ≥ 0 (اختياري)
```

**ot.php (بدء إضافي):**
```
token      → string, 64 حرف (مطلوب)
latitude   → numeric (مطلوب)
longitude  → numeric (مطلوب)
شرط:      → يوجد سجل 'out' اليوم
شرط:      → allow_overtime = 1
```

**overtime-end.php:**
```
token      → string, 64 حرف (مطلوب)
شرط:      → يوجد سجل 'overtime-start' اليوم
شرط:      → المدة ≥ overtime_min_duration
```

**verify-device.php:**
```
token       → string, 64 حرف (مطلوب)
fingerprint → string (مطلوب)
```

**submit-report.php:**
```
employee_id  → integer (مطلوب)
report_type  → string: violation|complaint|suggestion|other (مطلوب)
report_text  → string (مطلوب إذا لا صورة/صوت)
images       → file array, max 5, أنواع: jpg/png/gif (اختياري)
voice        → file, أنواع: webm/ogg/mp3 (اختياري)
```

**auth-pin.php:**
```
pin → string, 4 أرقام (مطلوب)
```

## نمط الاستجابات (Response Patterns)

### استجابة نجاح

```json
{
    "success": true,
    "message": "رسالة نجاح بالعربية",
    "data": { ... }   // بيانات إضافية (اختيارية)
}
```

### استجابة فشل

```json
{
    "success": false,
    "message": "رسالة خطأ بالعربية",
    "error": "error_code"  // كود خطأ (اختياري)
}
```

### أمثلة حقيقية

```json
// نجاح تسجيل الحضور
{
    "success": true,
    "message": "تم تسجيل الدخول بنجاح",
    "employee_name": "أحمد محمد",
    "timestamp": "2026-03-15 08:15:30",
    "distance": 10
}

// فشل — خارج النطاق
{
    "success": false,
    "message": "أنت خارج نطاق العمل! المسافة: 350 متر",
    "distance": 350
}

// خطأ في الطريقة
{
    "success": false,
    "message": "Method Not Allowed"
}
```

### الدالة المساعدة

```php
// تُستخدم في جميع نقاط API:
jsonResponse(['success' => true, 'message' => '...'], 200);
jsonResponse(['success' => false, 'message' => '...'], 400);
// ← تضبط Content-Type: application/json + تُنهي التنفيذ
```

## التخزين المؤقت (Caching)

### نمط الـ Static Cache في الإعدادات

```php
// includes/functions.php
function loadAllSettings(bool $refresh = false): array {
    static $cache = null;       // ← تُخزَّن في ذاكرة PHP لمدة الطلب
    if ($cache !== null && !$refresh) {
        return $cache;          // ← يُعاد الـ cache بدون استعلام DB
    }
    $cache = fetchAllFromDB();  // ← استعلام واحد: SELECT * FROM settings
    return $cache;
}
```

**كيف يعمل:**
```
الطلب الأول في نفس HTTP request:
  getSystemSetting('work_latitude')
  → loadAllSettings() → SELECT * FROM settings → $cache = [...]
  → يُعيد القيمة من $cache

الطلب الثاني في نفس HTTP request:
  getSystemSetting('geofence_radius')
  → loadAllSettings() → $cache موجود
  → يُعيد القيمة مباشرة بدون DB query
```

**التحديث:**
```php
setSystemSetting('key', 'value');
// → UPDATE settings SET ... WHERE setting_key = ?
// → loadAllSettings(refresh: true)  ← يُبطل الـ cache
```

> **ملاحظة:** هذا "Per-Request Cache" — يعيش فقط خلال طلب HTTP واحد. لا يُخزَّن بين الطلبات (لا Redis/Memcached).
