# نظام الحضور والانصراف — نظرة عامة

## ما هو النظام؟

نظام ويب متكامل لتسجيل حضور وانصراف الموظفين عبر الهاتف الذكي، يعتمد على:
- **الموقع الجغرافي (GPS)** — لا يمكن التسجيل إلا من داخل نطاق العمل
- **دعم الفروع المتعددة** — كل فرع له إحداثياته ونصف قطره الخاص
- **ورديات متعددة** — حتى 3 ورديات لكل فرع مع كشف تلقائي للوردية النشطة
- **رابط فريد لكل موظف** — بدلاً من كلمات المرور
- **نظام PIN** — رقم سري فريد (4 أرقام) لكل موظف
- **باركود QR** — توليد QR Code لبوابة الموظف مع شعار التطبيق
- **ربط الجهاز (3 أوضاع)** — حر / صارم (يمنع) / مراقبة صامتة (يسجل التلاعب بصمت)
- **تتبع ملكية الأجهزة** — النظام يعرف تلقائياً لمن ينتمي كل جهاز بناءً على الاستخدام
- **كشف التلاعب** — تسجيل بالنيابة، أجهزة مختلفة، مع سجل كامل
- **بلاغات سرية** — زر عائم للموظفين لإرسال بلاغات مع نص/صور/صوت (مع مؤثرات صوتية)
- **خلفية صور متحركة** — عرض شرائح صور كخلفية لصفحة الحضور
- **واجهة عربية** — تصميم برتقالي عصري يدعم RTL بالكامل
- **خريطة تفاعلية** — خريطة Leaflet تُظهر مركز العمل ونطاقه في واجهة الموظف
- **صوت رادار** — صوت سونار عند فتح الصفحة (Web Audio API)
- **شعار مخصص** — الشعار في كل مكان مع أنيميشن متعددة
- **تنبيه GPS مستمر** — نافذة منبثقة لتفعيل الموقع
- **PWA** — تطبيق ويب تقدمي مع Service Worker
- **Rate Limiting** — حماية من الطلبات المتكررة على API

## المصطلحات المهمة

| المصطلح | التعريف |
|---------|---------|
| **PIN** | رقم سري فريد من 4 أرقام يُعطى لكل موظف — يُستخدم لدخول بوابة الموظف |
| **Token** | نص عشوائي (64 حرف hex) يُمثّل رابط الموظف الفريد — بديل عن كلمة المرور |
| **Geofence** | السياج الجغرافي — نطاق دائري حول موقع العمل يُحدَّد بالمتر |
| **نافذة زمنية** | الفترة المسموح فيها بالتسجيل — من `shift_start - 60 دقيقة` حتى `shift_end` |
| **ربط الجهاز** | ربط بصمة جهاز الموظف بحسابه — 3 أوضاع: حر / صارم / مراقبة |
| **بصمة الجهاز (Fingerprint)** | قيمة SHA-256 تُحسب من خصائص المتصفح والجهاز — تُستخدم لتمييز الأجهزة |
| **الوردية (Shift)** | فترة عمل محددة ببداية ونهاية — كل فرع يدعم حتى 3 ورديات |
| **العمل الإضافي (Overtime)** | ساعات بعد الانصراف — يبدأ بعد فترة محددة وله حد أدنى |
| **التأخر (Late)** | الفرق بين `shift_start` ووقت تسجيل الحضور الفعلي — يُحسب بالدقائق |
| **Soft Delete** | أرشفة الموظف بدون حذف بياناته — يُستعاد بإزالة `deleted_at` |
| **Haversine** | صيغة رياضية لحساب المسافة بين نقطتين على سطح الكرة الأرضية |

## التقنيات المستخدمة

| المكوّن | التقنية |
|---------|---------|
| لغة الخادم | PHP 8.2+ |
| قاعدة البيانات | MySQL 8.0 (utf8mb4) |
| خادم الويب | Apache / LiteSpeed |
| الواجهة الأمامية | HTML5 + CSS3 + JavaScript (Vanilla) |
| الرسم البياني (رادار) | HTML5 Canvas API |
| الخرائط | Leaflet.js (واجهة الموظف + الإدارة) |
| الخطوط | Google Fonts — Tajawal |
| الموقع الجغرافي | Web Geolocation API + صيغة Haversine |
| التشفير | SHA-256 (SubtleCrypto) |
| الصوت | Web Audio API (صوت سونار) |
| PWA | Service Worker + manifest.json |

## تدفق العمل الأساسي

```
المشرف:
1. يضيف الموظف من لوحة الإدارة
2. يضبط إعدادات الموقع وأوقات الدوام
3. يرسل الرابط للموظف عبر واتساب

الموظف:
1. يضغط الرابط → صفحة الرادار تفتح (مع خلفية صور متحركة)
2. التحقق من بصمة الجهاز حسب الوضع (حر/صارم/مراقبة)
3. يصل للمكتب → يضغط "تسجيل الحضور"
4. في نهاية الدوام → يضغط "تسجيل الانصراف"
5. يمكنه إرسال بلاغ سري عبر الزر العائم (🛡️)
```
