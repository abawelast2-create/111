### 1.5 جدول `branch_shifts` — ورديات الفروع

| العمود | النوع | القيد | الوصف |
|--------|-------|-------|-------|
| `id` | INT | PK, AUTO_INCREMENT | معرّف |
| `branch_id` | INT | FK → branches.id | معرّف الفرع |
| `shift_number` | TINYINT | NOT NULL | رقم الوردية (1-3) |
| `shift_start` | TIME | NOT NULL | بداية الوردية |
| `shift_end` | TIME | NOT NULL | نهاية الوردية |
| `is_active` | TINYINT(1) | DEFAULT 1 | مفعّلة |
| `created_at` | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | تاريخ الإضافة |

**UNIQUE KEY:** `(branch_id, shift_number)` — لا يمكن تكرار نفس رقم الوردية لنفس الفرع.

**آلية الكشف التلقائي (`detectActiveShift()`):**
- يفحص الوقت الحالي مقابل جميع الورديات النشطة
- يدعم الورديات التي تعبر منتصف الليل (23:00 → 07:00)
- نافذة التسجيل: `shift_start - 60 دقيقة` حتى `shift_end`

**أوضاع ربط الجهاز (`device_bind_mode`):**
| القيمة | الوضع | السلوك |
|-------|------|--------|
| 0 | حر | الرابط يعمل من أي جهاز بدون ربط |
| 1 | صارم | يربط الجهاز ويمنع أي جهاز مختلف (شاشة قفل) |
| 2 | مراقبة صامتة | يسمح بالدخول لكن يسجل التلاعب بصمت |

**ملاحظة:** إذا كان للموظف `branch_id` محدد، يُستخدم نطاق الفرع الجغرافي. وإلا تُستخدم الإعدادات العامة.

### أولوية الإعدادات (Settings Priority)

النظام يتبع أولوية واضحة عند جلب الإعدادات:

| الأولوية | المصدر | مثال |
|----------|--------|------|
| 1 (الأعلى) | إعدادات الفرع (`branches`) | `geofence_radius`, `allow_overtime`, أوقات الورديات |
| 2 (الافتراضي) | الإعدادات العامة (`settings`) | `work_latitude`, `geofence_radius`, أوقات الدوام |

**القاعدة:** إذا الموظف مرتبط بفرع (`branch_id IS NOT NULL`) → يُستخدم إعداد الفرع.  
إذا لم يكن مرتبطاً بفرع → تُستخدم الإعدادات العامة.

```php
// مثال في isWithinGeofence():
if ($branchId) {
    $branch = getBranch($branchId);
    $lat = $branch['latitude'];
    $radius = $branch['geofence_radius'];
} else {
    $lat = getSystemSetting('work_latitude');
    $radius = getSystemSetting('geofence_radius');
}
```

## استعلامات شائعة مرجعية

```sql
-- إحصائيات اليوم
SELECT
  (SELECT COUNT(DISTINCT employee_id) FROM attendances WHERE DATE(timestamp) = CURDATE() AND type = 'in') AS checked_in,
  (SELECT COUNT(DISTINCT employee_id) FROM attendances WHERE DATE(timestamp) = CURDATE() AND type = 'out') AS checked_out,
  (SELECT COUNT(*) FROM employees WHERE is_active = 1) AS total;

-- موظفي فرع معين
SELECT e.*, b.name AS branch_name FROM employees e
LEFT JOIN branches b ON e.branch_id = b.id
WHERE b.id = ?;

-- حضور موظف معين اليوم
SELECT type, timestamp FROM attendances
WHERE employee_id = ? AND DATE(timestamp) = CURDATE()
ORDER BY timestamp ASC;

-- الغائبون اليوم
SELECT e.name, e.job_title, b.name AS branch_name FROM employees e
LEFT JOIN branches b ON e.branch_id = b.id
WHERE e.is_active = 1
AND e.id NOT IN (
  SELECT DISTINCT employee_id FROM attendances WHERE DATE(timestamp) = CURDATE()
);

-- تقرير موظف لفترة محددة
SELECT a.type, a.timestamp, a.latitude, a.longitude
FROM attendances a
WHERE a.employee_id = ? AND a.timestamp BETWEEN ? AND ?
ORDER BY a.timestamp;
```

## متى تتغير الخلايا؟ (Event → Column Map)

### جدول `employees`

| الحدث | الأعمدة المتأثرة |
|-------|-----------------|
| إنشاء موظف جديد | كل الأعمدة (`pin` + `unique_token` يُولَّدان تلقائياً) |
| أول دخول من جهاز (صارم/مراقبة) | `device_fingerprint`, `device_registered_at` |
| تعديل بيانات الموظف | `name`, `job_title`, `branch_id`, `phone`, `device_bind_mode` |
| رفع صورة بروفايل | `profile_photo` |
| تغيير PIN | `pin`, `pin_changed_at` |
| أرشفة الموظف | `deleted_at`, `unique_token` (يُعاد توليده) |
| استرجاع الموظف | `deleted_at` → NULL |
| إعادة توليد الرابط | `unique_token` |
| إعادة ضبط الجهاز | `device_fingerprint` → NULL, `device_registered_at` → NULL |

### جدول `attendances`

| الحدث | الأعمدة المتأثرة |
|-------|-----------------|
| تسجيل حضور | سجل جديد: `type='in'`, `late_minutes` يُحسب, `latitude/longitude`, `ip_address`, `user_agent` |
| تسجيل انصراف | سجل جديد: `type='out'`, `late_minutes=0` |
| بدء دوام إضافي | سجل جديد: `type='overtime-start'` |
| إنهاء دوام إضافي | سجل جديد: `type='overtime-end'` |
| خروج تلقائي (cron) | سجل جديد: `type='out'`, `notes='auto-checkout'` |

### جدول `branches`

| الحدث | الأعمدة المتأثرة |
|-------|-----------------|
| إنشاء فرع | كل الأعمدة |
| تعديل الموقع | `latitude`, `longitude`, `geofence_radius` |
| تعديل إعدادات الإضافي | `allow_overtime`, `overtime_start_after`, `overtime_min_duration` |
| تعطيل الفرع | `is_active` → 0 |

### جدول `known_devices`

| الحدث | الأعمدة المتأثرة |
|-------|-----------------|
| أول دخول من جهاز | سجل جديد: `fingerprint`, `employee_id`, `usage_count=1` |
| نفس الجهاز مرة أخرى | `usage_count` += 1, `last_used_at` |

### جدول `tampering_cases`

| الحدث | الأعمدة المتأثرة |
|-------|-----------------|
| جهاز مختلف (صارم/مراقبة) | سجل جديد: `case_type='different_device'`, `details_json` |
| تسجيل بالنيابة (نفس IP لموظفَين) | سجل جديد: `case_type='proxy_checkin'` |

### جدول `secret_reports`

| الحدث | الأعمدة المتأثرة |
|-------|-----------------|
| إرسال بلاغ | سجل جديد: `report_text`, `report_type`, صور/صوت |
| مراجعة البلاغ | `status`, `reviewed_at`, `reviewed_by` |
| إضافة ملاحظات | `admin_notes`, `status` |

### جدول `leaves`

| الحدث | الأعمدة المتأثرة |
|-------|-----------------|
| طلب إجازة | سجل جديد: `status='pending'` |
| موافقة | `status='approved'`, `approved_by` |
| رفض | `status='rejected'` |
